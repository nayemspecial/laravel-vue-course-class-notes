**প্রিয় শিক্ষার্থীবৃন্দ,**

**প্রযুক্তি প্লাস একাডেমির** পক্ষ থেকে আপনাদের সবাইকে স্বাগতম জানাচ্ছি। গত পর্বে আমরা আমাদের সাইডবারের মেনুগুলোকে সফলভাবে ডায়নামিক করেছিলাম। আজকের পর্বে আমরা আমাদের **নতুন ট্যাগ তৈরি করার (Tag Creation)** জন্য প্রয়োজনীয় রাউট, কন্ট্রোলার মেথড এবং ক্রিয়েট ফর্ম ভিউ ফাইল সেটআপ করে সম্পূর্ণ ডায়নামিক করার কাজ সম্পাদন করব।

আপনাদের বোঝার সুবিধার্থে রাউট ডেফিনেশন, কন্ট্রোলার লজিক এবং ব্লেড ফর্মে বৈ্যালিডেশন ও ওল্ড ডাটা হ্যান্ডলিং নিচে সুন্দরভাবে সাজিয়ে দেওয়া হলো:

---

## ১. রাউট ডিফাইন করা (`routes/web.php`)

নতুন ট্যাগ তৈরির ফর্ম প্রদর্শন করা এবং ফর্মে ডাটা সাবমিট করার জন্য আমাদের দুটি রাউট তৈরি করতে হবে। একটি **GET** রাউট (ফর্ম দেখানোর জন্য) এবং একটি **POST** রাউট (ডাটা প্রসেস বা স্টোর করার জন্য):

```php
use App\Http\Controllers\Admin\TagController;

// ট্যাগ তৈরির ফর্ম দেখানোর জন্য (GET Route)
Route::get('admin/tags/create', [TagController::class, 'create'])->name('admin.tags.create');

// ফর্মে সাবমিট করা ডাটা স্টোর করার জন্য (POST Route)
Route::post('admin/tags', [TagController::class, 'store'])->name('admin.tags.store');

```

## ২. কন্ট্রোলারে ক্রিয়েট মেথড যোগ করা (`TagController.php`)

এবার আমরা `TagController.php` ফাইলে গিয়ে `create()` মেথড তৈরি করব, যা আমাদের ক্রিয়েট ভিউ ফাইলটিকে রিটার্ন করবে:

```php
<?php

namespace App\Http\Controllers\Admin;

use App\Http\Controllers\Controller;
use Illuminate\Http\Request;

class TagController extends Controller
{
    // ট্যাগ তৈরির ভিউ ফাইল প্রদর্শন করা
    public function create()
    {
        return view('admin.tags.create');
    }
}

```

## ৩. ব্লেড ভিউ ফাইল তৈরি ও মাস্টারিং (`resources/views/admin/tags/create.blade.php`)

আমরা `resources/views/admin/tags/` ডিরেক্টরিতে `create.blade.php` নামে নতুন ফাইল তৈরি করব। এরপর আমাদের মাস্টার লেআউট এক্সটেন্ড করে প্রয়োজনীয় মেইন ফর্মের অংশটুকু সেকশনের মধ্যে বসিয়ে দেব:

```html
@extends('admin.layouts.app')

@section('content')
<div class="container mx-auto px-4 py-6">
    <div class="flex justify-between items-center mb-6">
        <h2 class="text-2xl font-bold text-gray-800">নতুন ট্যাগ তৈরি করুন</h2>
        <!-- ট্যাগ তালিকায় ফিরে যাওয়ার বাটন -->
        <a href="{{ route('admin.tags.index') }}" class="btn-secondary">
            ট্যাগ তালিকায় ফিরে যান
        </a>
    </div>

    <!-- ট্যাগ ফর্ম -->
    <form action="{{ route('admin.tags.store') }}" method="POST" class="bg-white p-6 rounded-lg shadow">
        @csrf

        <div class="mb-4">
            <label for="name" class="block text-gray-700 font-medium mb-2">ট্যাগের নাম</label>
            
            <!-- ইনপুট ফিল্ড ও ওল্ড ডাটা হ্যান্ডলিং -->
            <input type="text" 
                   name="name" 
                   id="name" 
                   value="{{ old('name') }}" 
                   class="field-input @error('name') is-invalid @enderror" 
                   placeholder="ট্যাগের নাম লিখুন...">

            <!-- বৈ্যালিডেশন এরর মেসেজ প্রদর্শনী -->
            @error('name')
                <div class="field-error text-red-500 text-sm mt-1">
                    {{ $message }}
                </div>
            @enderror
        </div>

        <div class="flex justify-end gap-2">
            <a href="{{ route('admin.tags.index') }}" class="btn-light">বাতিল করুন</a>
            <button type="submit" class="btn-primary">ট্যাগ সেভ করুন</button>
        </div>
    </form>
</div>
@endsection

```

## ৪. কোডের লজিক্যাল ব্যাখ্যা

* **`@csrf` ডিরেক্টিভ:** লারাভেলে প্রতিটি POST ফর্ম সাবমিশনের জন্য CSRF (Cross-Site Request Forgery) টোকেন ব্যবহার করা বাধ্যতামূলক, যা অ্যাপ্লিকেশনকে অনাকাঙ্ক্ষিত সিকিউরিটি থ্রেট থেকে সুরক্ষিত রাখে।
* **`value="{{ old('name') }}"`:** ফর্মে তথ্য সাবমিট করার পর কোনো কারণে যদি বৈ্যালিডেশন এরর আসে, তবে এই হেল্পার মেথডটির সাহায্যে ব্যবহারকারীর পূর্বে টাইপ করা ইনপুটটি ফিল্ডে স্বয়ংক্রিয়ভাবে থেকে যায়।


* **`@error('name')` ডিরেক্টিভ:** ট্যাগ নেমে কোনো ফর্মাল বা লজিক্যাল এরর থাকলে এটি সক্রিয় হয় এবং ইনপুট ফিল্ডে এরর ক্লাস যোগ করার পাশাপাশি সুনির্দিষ্ট মেসেজ ডায়নামিকালি প্রদর্শন করে।

## ৫. প্র্যাকটিক্যাল টেস্ট ও যাচাইকরণ

কোড সেভ করে ব্রাউজারে `admin/tags/create` ইউআরএলে প্রবেশ করলে আমাদের সাজানো ফর্মটি লেআউটের ভেতরে সঠিকভাবে প্রদর্শিত হবে। এখন ফর্মে কোনো ট্যাগের নাম টাইপ করে "ট্যাগ সেভ করুন" বাটনে ক্লিক করলে তা POST রিকোয়েস্ট আকারে কন্ট্রোলারের `store()` মেথডে চলে যাবে।

আজকের পর্বে আমরা ক্রিয়েট ফর্ম ভিউ ফাইল ও ইউআই সফলভাবে সেটআপ করতে পেরেছি। পরবর্তী পর্বে আমরা কন্ট্রোলারে আসা এই রিকোয়েস্ট রিসিভ করে ফর্ম বৈ্যালিডেশন এবং ডাটাবেজে ডাটা সংরক্ষণ করার লজিক নিয়ে কাজ করব।

সবাই মনোযোগ দিয়ে কোডগুলো প্র্যাকটিস করে ফেলুন। ধন্যবাদ!
