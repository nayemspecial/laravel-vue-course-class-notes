**প্লাটফর্ম:** প্রযুক্তিপ্লাস একাডেমী (ProjuktiPlus Academy)

**বিষয়:** ১০.৫ ট্যাগ ক্রিয়েট ফর্মে অ্যাকশন মেথড ও CSRF টোকেন যুক্ত করা, ক্রিয়েট ফর্ম থেকে সাবমিট করা ডাটা কন্ট্রোলারে গ্রহণ করা, ডাটাবেজে নতুন ট্যাগ সংরক্ষণ করা, ইনপুট ডাটা ভ্যালিডেশন, স্লাগ জেনারেট করা, ডাটাবেজে সংরক্ষণ করে সাকসেস মেসেজসহ রিডাইরেক্ট করা।

**প্রযুক্তি প্লাস একাডেমির** পক্ষ থেকে আপনাদের সবাইকে স্বাগতম জানাচ্ছি। গত পর্বে আমরা আমাদের ক্রিয়েট ফর্মের ইউআই ডিজাইন এবং প্রাথমিক সেটআপ সম্পন্ন করেছিলাম। আজকের পর্বে আমরা কীভাবে ক্রিয়েট ফর্ম থেকে সাবমিট করা ডাটা কন্ট্রোলারে গ্রহণ করব, ভ্যালিডেশন সম্পন্ন করব, স্লাগ জেনারেট করব এবং ডাটাবেজে সংরক্ষণ (Store) করে সাকসেস মেসেজসহ রিডাইরেক্ট করব—তার পুরো প্র্যাকটিক্যাল লজিক শিখব।

নিচে ধাপভিত্তিক সম্পূর্ণ কোড এবং বিস্তারিত ব্যাখ্যা দেওয়া হলো:

## ১. ক্রিয়েট ফর্মে অ্যাকশন, মেথড ও CSRF টোকেন যুক্ত করা

ডাটা সাবমিট করার জন্য `resources/views/admin/tags/create.blade.php` ফাইলের `<form>` ট্যাগে অ্যাকশন, পোস্ট মেথড এবং `@csrf` টোকেন যুক্ত করতে হবে:

```html
<form action="{{ route('admin.tags.store') }}" method="POST" class="bg-white p-6 rounded-lg shadow">
    @csrf
    
    <!-- ইনপুট ফিল্ড ও অন্যান্য কোড -->
</form>

```

## ২. কন্ট্রোলারে স্টোর মেথড তৈরি করা (`TagController.php`)

এবার আমাদের `TagController.php` ফাইলে প্রয়োজনীয় হেল্পার ক্লাস যুক্ত করতে হবে এবং `store()` মেথডে ভ্যালিডেশন, স্লাগ জেনারেশন ও ডাটা সেভ করার লজিক লিখতে হবে:

```php
<?php

namespace App\Http\Controllers\Admin;

use App\Http\Controllers\Controller;
use App\Models\Tag;
use Illuminate\Http\Request;
use Illuminate\Support\Str; // স্লাগ তৈরির জন্য Str হেল্পার ক্লাস

class TagController extends Controller
{
    // ... অন্যান্য মেথড

    // ডাটাবেজে নতুন ট্যাগ সংরক্ষণ করা
    public function store(Request $request)
    {
        // ১. ইনপুট ডাটা ভ্যালিডেশন
        $validated = $request->validate([
            'name' => 'required|string|max:255|unique:tags,name',
        ]);

        // ২. ডাটাবেজে ট্যাগ তৈরি ও স্লাগ জেনারেশন
        Tag::create([
            'name' => $validated['name'],
            'slug' => Str::slug($validated['name']), // নাম থেকে ডায়নামিক স্লাগ তৈরি
        ]);

        // ৩. ট্যাগ লিস্ট পেজে সাকসেস মেসেজসহ রিডাইরেক্ট
        return redirect()->route('admin.tags.index')->with('success', 'ট্যাগ সফলভাবে তৈরি হয়েছে।');
    }
}

```

## ৩. ট্যাগ লিস্ট পেজে "নতুন ট্যাগ" বাটন লিঙ্ক করা (`index.blade.php`)

আমাদের ট্যাগ লিস্ট পেজের (`index.blade.php`) "নতুন ট্যাগ তৈরি করুন" বাটনটিতে ক্রিয়েট ফর্মের রাউটটি বসিয়ে দিতে হবে, যেন সেখানে ক্লিক করলে ব্যবহারকারী ক্রিয়েট ফর্মে যেতে পারেন:

```html
<a href="{{ route('admin.tags.create') }}" class="btn-primary">
    নতুন ট্যাগ তৈরি করুন
</a>

```

## ৪. কোডের লজিক্যাল ব্যাখ্যা

* **`Request $request`:** এইচটিটিপি রিকোয়েস্টের মাধ্যমে আসা সমস্ত ইনপুট ডাটা রিসিভ করতে লারাভেলের `Request` অবজেক্ট ডিপেন্ডেন্সি ইনজেকশন হিসেবে কাজ করে।


* **`$request->validate()`:** ট্যাগের নাম ফিল্ডটি যেন ফাঁকা না থাকে (`required`), স্ট্রিং হয় (`string`), সর্বোচ্চ ২৫৫ ক্যারেক্টার হয় (`max:255`) এবং `tags` টেবিলের `name` কলামে আগে থেকে না থাকে (`unique`) তা নিশ্চিত করে।


* **`Str::slug()`:** এটি লারাভেলের `Illuminate\Support\Str` ক্লাসের একটি মেথড। এটি ট্যাগের নামকে (যেমন: "Laravel Web Development") ইউআরএল-ফ্রেন্ডলি ফরম্যাটে (যেমন: `laravel-web-development`) রূপান্তরিত করে।


* **`with('success', '...')`:** ডাটা সফলভাবে সেভ হওয়ার পর ইউজারকে সেশন মেসেজ (Session Message) বা নোটিফিকেশন পাঠানোর জন্য এটি ব্যবহার করা হয়।

## ৫. ব্রাউজারে আউটপুট টেস্ট

কোড সেভ করে ব্রাউজারে গিয়ে ট্যাগ তালিকায় "নতুন ট্যাগ তৈরি করুন" বাটনে ক্লিক করুন। ফর্মে নতুন ট্যাগের নাম (যেমন: `JavaScript`) লিখে "ট্যাগ সেভ করুন" বাটনে ক্লিক করলে তা ডাটাবেজে সেভ হবে এবং অটোমেটিক স্লাগ জেনারেট হয়ে আপনাকে পুনরায় ট্যাগ লিস্ট পেজে নিয়ে আসবে।

আমাদের ট্যাগ তৈরির সম্পূর্ণ ফাংশনালিটি সফলভাবে সম্পন্ন হয়েছে। পরবর্তী পর্বে আমরা ডাটাবেজে সেভ হওয়া ট্যাগগুলোর জন্য সাকসেস মেসেজ ডিসপ্লে করা এবং ডাটা এডিট, আপডেট ও ডিলিট করার ফিচার নিয়ে কাজ করব।

সবাই মনোযোগ দিয়ে প্র্যাকটিস করুন। ধন্যবাদ!
