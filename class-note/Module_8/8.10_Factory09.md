# লারাভেল ইউজার সিডার (User Seeder) তৈরি ও কাস্টম ডেটা সিডিং

**প্লাটফর্ম:** প্রজু্ক্তিপ্লাস একাডেমী (ProjuktiPlus Academy)

**বিষয়:** ৮.১০ ডিফল্ট ডাটাবেজ সিডার রিভিউ, কাস্টম ইউজার সিডার ফাইল তৈরি করা, ইউজার মডেল ক্লাস ইম্পোর্ট করা, ইউজার রোল সিডিং কোড সেট করা, সিডার রানের পর টার্মিনালে কাস্টম মেসেজ দেখানো। 

এই পর্ব থেকে আমরা লারাভেলের সিডার (Seeder) সম্পর্কিত কোড এবং কাজ শুরু করবো। সিডার মূলত ডাটাবেজে রিয়েলিস্টিক ডামি ডেটা পুশ বা ইনসার্ট করার জন্য ব্যবহার করা হয়।

শুরুতেই আমরা ডাটাবেজ ফোল্ডারের ভেতর থেকে মাইগ্রেশন এবং ফ্যাক্টরি ফাইলগুলো ক্লোজ করে নেবো। এরপর সিডার ফোল্ডারে গেলে দেখতে পাবো ডিফল্টভাবে `DatabaseSeeder.php` ফাইলটি ক্রিয়েট করা আছে।

---

### ১. ডিফল্ট ডাটাবেজ সিডার রিভিউ

লারাভেল ইনস্টল করার সাথে সাথে `DatabaseSeeder.php` ফাইলের ভেতরে কিছু কোড ডিফল্ট বা অটোমেটিক দেওয়া থাকে। যেমন:

```php
// User::factory()->create([
//     'name' => 'Test User',
//     'email' => 'test@example.com',
// ]);

```

কমিউনিটিতে বা বড় প্রজেক্টে কন্ট্রিবিউশন করার সময় এই জিনিসটা খুব কাজে দেয়। কোনো সিনিয়র ডেভেলপার প্রজেক্ট রান করার পর যদি সিস্টেমে লগইন করার প্রয়োজন ফিল করেন, তখন তারা এই ডিফল্ট ইউজার বা ইমেইল এবং পাসওয়ার্ড হিসেবে `password` ব্যবহার করে লগইন করতে পারেন। এছাড়া সেখানে ১০টি রেন্ডম ইউজার তৈরির কোডও কমেন্ট আউট করা থাকে। প্রয়োজন হলে কমেন্ট উঠিয়ে দিয়ে সেটি রান করা যায়।

---

### ২. কাস্টম ইউজার সিডার ফাইল তৈরি করা

আমাদের প্রজেক্টের রিকোয়ারমেন্ট অনুযায়ী একটি ডেডিকেটেড ইউজার সিডার প্রয়োজন। নতুন সিডার ফাইল ক্রিয়েট করার জন্য টার্মিনালে নিচের আর্টিজান কমান্ডটি রান করবো:

```bash
php artisan make:seeder UserSeeder

```

*নোট: কমান্ড লেখার সময় ক্লাসের নাম PascalCase অর্থাৎ `UserSeeder` (U এবং S ক্যাপিটাল) এভাবে লিখবেন।*

কমান্ডটি সফলভাবে রান হলে `database/seeders` ডিরেক্টরির মধ্যে `UserSeeder.php` নামে একটি নতুন ফাইল চলে আসবে। ফাইলটি ওপেন করলে আমরা সিডারের বেসিক স্ট্রাকচার বা কোডগুলো দেখতে পাবো।

---

### ৩. মডেল ইম্পোর্ট করা (Class Import)

যেহেতু আমরা এই সিডার ফাইলের ভেতরে ইউজার মডেল এবং তার ফ্যাক্টরি নিয়ে কাজ করবো, তাই ফাইলের একদম উপরে নেমস্পেসের নিচে অবশ্যই `User` মডেলের ক্লাসটি ইম্পোর্ট করে নিতে হবে:

```php
use App\Models\User;

```

---

### ৪. `run()` মেথডের ভেতরে ইউজার রোল ভিত্তিক সিডিং লজিক

আমরা আমাদের প্রজেক্টে বিভিন্ন রোলের ইউজার জেনারেট করবো। ইউজার ফ্যাক্টরিতে আমাদের যে কাস্টম স্টেটগুলো (যেমন: `superAdmin`, `admin`, `author`, `reader`) তৈরি করা ছিল, সেগুলোকে ব্যবহার করে আমরা নির্দিষ্ট সংখ্যায় ইউজার তৈরি করবো। কাউন্ট উল্লেখ না করলে ডিফল্টভাবে ১টি করে রেকর্ড তৈরি হয়।

চলুন `UserSeeder` ফাইলের `run()` মেথডের ভেতরে কোডগুলো লিখে ফেলি:

#### ক) সুপার এডমিন ক্রিয়েট করা (১ জন)

```php
// ১ জন সুপার এডমিন রোল সহ ইউজার তৈরি হবে
User::factory()->superAdmin()->create();

```

#### খ) এডমিন ক্রিয়েট করা (২ জন)

এখানে ১টির বেশি ইউজার তৈরির জন্য আমরা স্টেট মেথডের পর লারাভেলের চেইনিং মেথড `count()` ব্যবহার করবো:

```php
// ২ জন এডমিন ইউজার তৈরি হবে
User::factory()->admin()->count(2)->create();

```

#### গ) অথর ক্রিয়েট করা (৫ জন)

```php
// ৫ জন অথর ইউজার তৈরি হবে
User::factory()->author()->count(5)->create();

```

#### ঘ) রিডার ক্রিয়েট করা (৫০ জন)

```php
// ৫০ জন রিডার ইউজার তৈরি হবে
User::factory()->reader()->count(50)->create();

```

---

### ৫. সিডার রানের পর টার্মিনালে কাস্টম মেসেজ বা ইনফো দেখানো

ইউজার সিডারটি পারফেক্টলি রান হওয়ার পর টার্মিনালে যেন একটি সুন্দর আউটপুট বা কনফার্মেশন মেসেজ শো করে, সেজন্য আমরা লারাভেলের `$this->command->info()` মেথড ব্যবহার করতে পারি। এর মাধ্যমে কতজন ইউজার ক্রিয়েট হলো তা পরিষ্কারভাবে কনসোলে দেখা যাবে:

```php
$this->command->info('User seeded successfully: 1 Super Admin, 2 Admins, 5 Authors, and 50 Readers created.');

```

---

### ৬. ইউজার সিডার ফাইলের সম্পূর্ণ কোড রূপরেখা

সব লজিক সাজানোর পর আমাদের `UserSeeder.php` ফাইলের সম্পূর্ণ কোড স্ট্রাকচারটি নিচে দেওয়া হলো:

```php
<?php

namespace Database\Seeders;

use App\Models\User;
use Illuminate\Database\Seeder;

class UserSeeder extends Seeder
{
    /**
     * Run the database seeds.
     */
    public function run(): void
    {
        // ১ জন সুপার এডমিন ক্রিয়েট করা হলো
        User::factory()->superAdmin()->create();

        // ২ জন এডমিন ক্রিয়েট করা হলো
        User::factory()->admin()->count(2)->create();

        // ৫ জন অথর ক্রিয়েট করা হলো
        User::factory()->author()->count(5)->create();

        // ৫০ জন রিডার ক্রিয়েট করা হলো
        User::factory()->reader()->count(50)->create();

        // টার্মিনালে সাকসেস মেসেজ প্রিন্ট করা
        $this->command->info('User seeded successfully: 1 Super Admin, 2 Admins, 5 Authors, and 50 Readers created.');
    }
}

```

---

### সারাংশ ও পরবর্তী পরিকল্পনা

এই পর্বে আমরা সাকসেসফুলি `UserSeeder` ফাইলের কাজ শেষ করলাম এবং রোল ভিত্তিক ইউজার ডেটা মেইনটেইন করা শিখলাম। আমাদের ডামি ডেটা জেনারেশনে ফেকার ডাটা এবং সিডার দুজনেই চমৎকারভাবে হেল্প করে।

ইউজার সিডারের কাজ আমরা এখানেই সেভ দিয়ে শেষ করছি। যেভাবে আমরা ইউজারের জন্য সিডার মেথডগুলো রেডি করলাম, ঠিক একইভাবে পরবর্তী পর্বে আমরা আমাদের প্রজেক্টের অন্যতম গুরুত্বপূর্ণ অংশ **ক্যাটাগরি সিডার (Category Seeder)** নিয়ে কাজ শুরু করবো।

আজকে এই পর্যন্তই, পরবর্তী পর্বে আবার দেখা হবে!
