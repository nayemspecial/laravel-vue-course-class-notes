# লারাভেল ক্যাটাগরি ও ট্যাগ সিডার (Category & Tag Seeder) তৈরি 

**প্লাটফর্ম:** প্রযুক্তিপ্লাস একাডেমী (ProjuktiPlus Academy)

**বিষয়:** ৮.১১ ক্যাটাগরি সিডার তৈরি, ক্যাটাগরি মডেল ক্লাস ইম্পোর্ট করা, run() মেথডের ভেতর ক্যাটাগরি তৈরি ও কাস্টম ইনফো মেসেজ সেটআপ, ট্যাগ সিডার ফাইল তৈরি, সিডার রানের পর টার্মিনালে কাস্টম মেসেজ দেখানো।

এই পর্বে আমরা আমাদের প্রজেক্টের **ক্যাটাগরি সিডার (Category Seeder)** এবং **ট্যাগ সিডার (Tag Seeder)** তৈরি করে তার ভেতরে প্রপার কোড ইমপ্লিমেন্টেশন শিখবো। নিচে পুরো প্রসেসটি নিখুঁত ও বিস্তারিত ক্লাস নোট আকারে ধাপে ধাপে দেওয়া হলো:

---

### ১. ক্যাটাগরি সিডার (Category Seeder) তৈরি ও সেটআপ

ক্যাটাগরি সিডার ফাইলটি তৈরি করার জন্য আমরা প্রথমে আমাদের টার্মিনালে যাবো এবং নিচের আর্টিজান কমান্ডটি রান করবো:

```bash
php artisan make:seeder CategorySeeder

```

কমান্ডটি সফলভাবে রান হওয়ার পর আমরা `database/seeders` ফোল্ডারের মধ্যে `CategorySeeder.php` ফাইলটি পেয়ে যাবো।

#### ক) মডেল ক্লাস ইম্পোর্ট করা

ক্যাটাগরি সিডারের ভেতরে যেহেতু আমরা `Category` মডেল ব্যবহার করবো, তাই ফাইলের একদম উপরে নেমস্পেসের নিচে অবশ্যই মডেলটি ইম্পোর্ট করে নিতে হবে। তা না হলে ডিফাইন করা ক্লাসটি লারাভেল বুঝতে পারবে না এবং এরর আসবে:

```php
use App\Models\Category;

```

#### খ) `run()` মেথডের ভেতর ক্যাটাগরি তৈরি ও কাস্টম ইনফো মেসেজ সেটআপ

আমরা আমাদের প্রজেক্টের জন্য ১০টি ক্যাটাগরি জেনারেট করতে চাই। এর জন্য আমরা `count(10)` ব্যবহার করে ক্যাটাগরি ক্রিয়েট করবো।

পাশাপাশি, সিডারটি পারফেক্টলি রান হওয়ার পর টার্মিনালে যেন একটি সুন্দর আউটপুট বা কনফার্মেশন মেসেজ শো করে, সেজন্য আমরা ইউজার সিডারের মতো এখানেও `$this->command->info()` মেথড ব্যবহার করবো। এটি আমাদের টার্মিনালে চমৎকার একটি ভিজ্যুয়াল ফিডব্যাক দেবে:

```php
    /**
     * Run the database seeds.
     */
    public function run(): void
    {
        // ১০টি ক্যাটাগরি তৈরি করা হলো
        Category::factory()->count(10)->create();

        // টার্মিনালে সাকসেস মেসেজ প্রিন্ট করা
        $this->command->info('Category seeded successfully: 10 dynamic categories created with unique Bengali names.');
    }

```

### ২. ট্যাগ সিডার (Tag Seeder) তৈরি ও সেটআপ

ক্যাটাগরির কাজ শেষ করার পর এবার আমরা **ট্যাগ সিডার** নিয়ে কাজ শুরু করবো।

*নোট: লারাভেলে ফ্যাক্টরি তৈরি করার সময় আমরা `--model` বা মডেলের ফ্ল্যাগ দিয়ে সহজেই এলাইন করে নিতে পারি। কিন্তু সিডার ফাইল ক্রিয়েট করার সময় লারাভেলের ডিফল্ট কমান্ডে মডেল এলাইন করার ফ্ল্যাগটি কিছুটা আলাদা বা সরাসরি কাজ নাও করতে পারে। তাই আমরা কোনো ফ্ল্যাগ না দিয়ে নরমাল কমান্ড রান করে ভেতরে হাতে লিখে মডেলটি এলাইন করে দেবো।*

টার্মিনালে ট্যাগ সিডার তৈরির কমান্ড:

```bash
php artisan make:seeder TagSeeder

```

#### ক) মডেল ক্লাস ইম্পোর্ট করা

একইভাবে আমরা ফাইলের উপরে আমাদের `Tag` মডেলের নেমস্পেস বা ক্লাসটি ইম্পোর্ট করে নেবো:

```php
use App\Models\Tag;

```

#### খ) `run()` মেথডের ভেতর ট্যাগ তৈরির লজিক

আমরা প্রজেক্টের ডামি ডাটা সেটআপের জন্য মোট ৩০টি ইউনিক ট্যাগ জেনারেট করতে চাই। আমাদের ট্যাগগুলো যেন ইউনিক হয়, সেই লজিক আমরা অলরেডি ফ্যাক্টরি ফাইলে বলে এসেছি। তাই এখানে শুধু `count(30)` দিয়ে ক্রিয়েট করে দিলেই হবে:

```php
    /**
     * Run the database seeds.
     */
    public function run(): void
    {
        // ৩০টি ট্যাগ তৈরি করা হলো
        Tag::factory()->count(30)->create();

        // টার্মিনালে সাকসেস মেসেজ প্রিন্ট করা
        $this->command->info('Tag seeded successfully: 30 unique tags created.');
    }

```

### ৩. সম্পূর্ণ কোড রূপরেখা (Complete Seeder Codes)

আপনাদের সুবিধার জন্য দুটি সিডার ফাইলের সম্পূর্ণ কোড নিচে দেওয়া হলো যেন স্ট্রাকচার বুঝতে কোনো ভুল না হয়:

#### ক্যাটাগরি সিডার ফাইল (`CategorySeeder.php`)

```php
<?php

namespace Database\Seeders;

use App\Models\Category;
use Illuminate\Database\Seeder;

class CategorySeeder extends Seeder
{
    /**
     * Run the database seeds.
     */
    public function run(): void
    {
        // ১০টি ক্যাটাগরি তৈরি
        Category::factory()->count(10)->create();

        // টার্মিনালে সাকসেস মেসেজ শো করা
        $this->command->info('Category seeded successfully: 10 dynamic categories created with unique Bengali names.');
    }
}

```

#### ট্যাগ সিডার ফাইল (`TagSeeder.php`)

```php
<?php

namespace Database\Seeders;

use App\Models\Tag;
use Illuminate\Database\Seeder;

class TagSeeder extends Seeder
{
    /**
     * Run the database seeds.
     */
    public function run(): void
    {
        // ৩০টি ট্যাগ তৈরি
        Tag::factory()->count(30)->create();

        // টার্মিনালে সাকসেস মেসেজ শো করা
        $this->command->info('Tag seeded successfully: 30 unique tags created.');
    }
}

```

### সারাংশ ও পরবর্তী পরিকল্পনা

এই পর্বে আমরা সফলভাবে আমাদের প্রজেক্টের জন্য `CategorySeeder` এবং `TagSeeder` তৈরির কাজ সম্পন্ন করলাম। যখন সিডার কমান্ডগুলো রান করা হবে, তখন আমাদের টার্মিনালে এই কাস্টম মেসেজগুলো সুন্দরভাবে ভেসে উঠবে এবং আমরা কনফার্ম হতে পারবো যে আমাদের সিডিংস ঠিকঠাক মতো কাজ করেছে।

আমাদের ক্যাটাগরি এবং ট্যাগের কাজ এখানেই শেষ। এর পর আমরা আমাদের প্রজেক্টের সবচেয়ে গুরুত্বপূর্ণ অংশ **পোস্ট সিডার (Post Seeder)** নিয়ে কাজ শুরু করবো। পোস্ট সিডারের মধ্যে রিলেশনশিপ, অ্যাটাচমেন্টসহ আমাদের বেশ কিছু অতিরিক্ত কোড লিখতে হবে এবং লজিকগুলো একটু বড় হবে।

তাই পোস্ট সিডারের সম্পূর্ণ কাজ নিখুঁতভাবে বোঝার জন্য আমরা এই পর্বটি এখানেই শেষ করছি এবং আগামী পর্বে পোস্ট সিডার নিয়ে বিস্তারিত কাজ শুরু করবো।
