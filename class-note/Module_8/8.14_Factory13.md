# লারাভেল কমেন্ট সিডার (Comment Seeder) ও নেস্টেড লুপের মাধ্যমে চাইল্ড রিপ্লাই ও পেন্ডিং কমেন্ট সিডিং

**প্লাটফর্ম:** প্রযুক্তিপ্লাস একাডেমী (ProjuktiPlus Academy)

**বিষয়:** ৮.১৪ নেস্টেড ফরইচ লুপের মাধ্যমে চাইল্ড রিপ্লাই জেনারেশন, ৩০% কমেন্টে রিপ্লাই দেওয়ার কন্ডিশন সেটআপ, রিপ্লাই কাউন্ট এবং ফ্যাক্টরি মেথড চেইনিং, প্রথম লুপের ভেতরে পেন্ডিং কমেন্ট সেটআপ, ২০% পেন্ডিং কমেন্ট নির্ধারণের লজিক সেট করা। 

এই পর্বে আমরা কমেন্ট সিডারের কাজকে আরও এক ধাপ এগিয়ে নিয়ে যাবো। গত পর্বে আমরা প্রতিটি পোস্টের জন্য এপ্রুভড প্যারেন্ট কমেন্ট তৈরি করা শিখেছিলাম। এই পর্বে আমরা শিখবো কীভাবে নেস্টেড ফরইচ (Nested Foreach) লুপ ব্যবহার করে সেই প্যারেন্ট কমেন্টগুলোর আন্ডারে চাইল্ড কমেন্ট বা রিপ্লাই (Reply) জেনারেট করতে হয় এবং কীভাবে নির্দিষ্ট পার্সেন্টেজে পেন্ডিং কমেন্ট (Pending Comment) লজিক সেটআপ করতে হয়।

নিচে পুরো প্রসেসটি নিখুঁত ও বিস্তারিত ক্লাস নোট আকারে ধাপে ধাপে দেওয়া হলো:

---

### ১. নেস্টেড ফরইচ (Nested Foreach) লুপের মাধ্যমে চাইল্ড রিপ্লাই জেনারেশন

একটি কমেন্টের ভেতর যেমন চাইল্ড কমেন্ট বা রিপ্লাই থাকে, ঠিক সেই স্ট্রাকচারটি ডামি ডেটায় ফুটিয়ে তোলার জন্য আমাদের লুপের ভেতর আরেকটি লুপ অর্থাৎ নেস্টেড লুপ ব্যবহার করতে হবে। ক্রিয়েট রিপ্লাই ফর সাম প্যারেন্টস কমেন্ট—এটি করার মূল উদ্দেশ্য হলো আমাদের ডামি ডেটা যেন রিয়ালিস্টিক দেখায়।

প্রথম `foreach` লুপের ভেতরে (যেখানে প্যারেন্ট কমেন্টগুলো তৈরি হচ্ছে), আমরা নতুন আরেকটি লুপ এভাবে শুরু করবো:

```php
// লুপের ভেতরের দ্বিতীয় ফরইচ লুপ
foreach ($parentComments as $parentComment) {
    // এখানে সকল প্যারেন্ট কমেন্টস আসবে এবং তার মধ্য থেকে একটা একটা করে কমেন্ট নিয়ে আমরা কাজ করবো
}

```

#### ক) ৩০% কমেন্টে রিপ্লাই দেওয়ার কন্ডিশন সেটআপ

বাস্তব সম্মত ডেটা তৈরির জন্য আমরা চাইলেই সব প্যারেন্ট কমেন্টের রিপ্লাই দেবো না। আমরা ধরে নিচ্ছি আনুমানিক **30%** প্যারেন্ট কমেন্টের রিপ্লাই বা চাইল্ড কমেন্ট থাকবে। এই ৩০% এর লজিক বা কন্ডিশনটি আমরা পিএইচপির `rand(1, 100)` ফাংশন দিয়ে এভাবে হ্যান্ডেল করতে পারি:

```php
// ১ থেকে ১০০ এর মধ্যে রেন্ডম সংখ্যা যদি ৩০ এর সমান বা ছোট হয়, তবেই রিপ্লাই তৈরি হবে (৩০% চান্স)
if (rand(1, 100) <= 30) {
    // রিপ্লাই জেনারেশনের কোড এখানে হবে
}

```

#### খ) রিপ্লাই কাউন্ট এবং ফ্যাক্টরি মেথড চেইনিং

যদি কন্ডিশন সত্য হয়, তবে প্রতিটি প্যারেন্ট কমেন্টের আন্ডারে আমরা বেশি রিপ্লাই দেবো না; অল্প করে অর্থাৎ **১ থেকে ৩টি** রিপ্লাই তৈরি করবো। রিপ্লাইগুলো যেন এপ্রুভড হয় এবং কমেন্ট ফ্যাক্টরির `reply` স্টেটের প্যারেন্ট আইডি যেন প্রপারলি বসে, সেজন্য মেথড চেইনিং নিচের মতো হবে:

```php
$replyCount = rand(1, 3);

Comment::factory()
    ->approved()
    ->reply()
    ->count($replyCount)
    ->create([
        'post_id'   => $post->id,
        'parent_id' => $parentComment->id, // লুপের ভেতরের সিঙ্গেল প্যারেন্ট কমেন্টের আইডি (ভুল করে আবার $parentComments লিখে ফেলবেন না)
    ]);

```

*নোট: লারাভেলের সিনট্যাক্স আমরা যতই গভীর বা চেইন করি না কেন, এটি কিন্তু চমৎকার রিডেবল (Readable) এবং দেখতেও বেশ প্রফেশনাল লাগে। ক্রিয়েট মেথডের ভেতরে অ্যাসোসিয়েটিভ অ্যারে ব্যবহার করে আমরা একই পোস্টের আইডি এবং নির্দিষ্ট প্যারেন্ট কমেন্টের আইডি সুন্দর করে কি এবং ভ্যালু আকারে পাস করে দিয়েছি।*

### ২. প্রথম লুপের ভেতরে পেন্ডিং কমেন্ট (Pending Comment) সেটআপ

চাইল্ড রিপ্লাইয়ের কাজ শেষ হওয়ার পর আমাদের পেন্ডিং কমেন্ট নিয়ে কাজ করতে হবে। যেহেতু পেন্ডিং কমেন্টের কোনো চাইল্ড বা ডাবল স্টেপ রিপ্লাইয়ের প্রয়োজন নেই, তাই এটি হবে একটি সিঙ্গেল স্টেপ কাজ। এর জন্য আমাদের সেকেন্ড লেভেলের যে ফরইচ (`foreach`) লুপটি আছে, সেটি থেকে বের হয়ে যেতে হবে এবং প্রথম বা মেইন ফরইচ লুপের ভেতরে কোড লিখতে হবে।

#### ক) ২০% পেন্ডিং কমেন্ট নির্ধারণের ম্যাথমেটিক্যাল লজিক

আমরা চাই মোট যতগুলো প্যারেন্ট কমেন্ট তৈরি হয়েছে, তার **20%** কমেন্ট যেন পেন্ডিং অবস্থায় থাকে এবং কমপক্ষে ১টি কমেন্ট যেন অবশ্যই পেন্ডিং হয়। পিএইচপির `max()` এবং `floor()` ফাংশন ব্যবহার করে এই ভগ্নাংশ বা পার্সেন্টেজের হিসাবটি নিচে দেখানো হলো:

```php
// টোটাল প্যারেন্ট কমেন্ট কাউন্টের সাথে ০.২ গুণ করে ২০% বের করা হলো এবং কমপক্ষে ১টি পেন্ডিং রাখা নিশ্চিত করা হলো
$pendingCount = max(1, floor($parentCommentCount * 0.2));

```

*(নোট: যারা ফ্লোর বা ম্যাক্স ফাংশনের কাজ ভুলে গেছেন, তারা পিএইচপির বেসিক মেথডগুলো আরেকবার দেখে নিতে পারেন। এটি ডাউন-রাউন্ডিং এবং সর্বোচ্চ মানটি লুফে নিতে সাহায্য করে।)*

#### খ) পেন্ডিং কমেন্ট ফ্যাক্টরি রান

পেন্ডিং কমেন্ট তৈরি করার সময় আমরা ফ্যাক্টরির ডিফল্ট স্টেট ব্যবহার করবো (যেহেতু ডিফল্টভাবে কমেন্ট পেন্ডিং থাকে) এবং শুধু পোস্ট আইডি ওভাররাইট করে দেবো:

```php
Comment::factory()
    ->count($pendingCount)
    ->create([
        'post_id' => $post->id,
    ]);

```

### ৩. কমেন্ট সিডার ফাইলের সম্পূর্ণ কোড রূপরেখা (Complete `CommentSeeder.php`)

সবগুলো লজিক, নেস্টেড লুপ এবং পার্সেন্টেজ ক্যালকুলেশন সাজানোর পর আমাদের কমেন্ট সিডার ফাইলটি দেখতে হুবহু এইরকম হবে:

```php
<?php

namespace Database\Seeders;

use App\Models\Post;
use App\Models\Comment;
use Illuminate\Database\Seeder;

class CommentSeeder extends Seeder
{
    /**
     * Run the database seeds.
     */
    public function run(): void
    {
        // সকল পোস্ট তুলে আনা হলো
        $posts = Post::all();

        // প্রথম লেভেলের লুপ (প্রতিটি পোস্টের জন্য)
        foreach ($posts as $post) {
            
            // ৫ থেকে ১৫টি রেন্ডম এপ্রুভড প্যারেন্ট কমেন্ট সংখ্যা নির্ধারণ
            $parentCommentCount = rand(5, 15);

            // প্যারেন্ট কমেন্ট তৈরি এবং ভেরিয়েবলে কালেকশন রাখা
            $parentComments = Comment::factory()
                ->approved()
                ->count($parentCommentCount)
                ->create([
                    'post_id' => $post->id,
                ]);

            // দ্বিতীয় লেভেলের লুপ (নেস্টেড লুপ - প্রতিটি প্যারেন্ট কমেন্টের জন্য রিপ্লাই তৈরি)
            foreach ($parentComments as $parentComment) {
                
                // ৩০% সম্ভাবনা বা কন্ডিশনে চাইল্ড রিপ্লাই তৈরি হবে
                if (rand(1, 100) <= 30) {
                    $replyCount = rand(1, 3);

                    Comment::factory()
                        ->approved()
                        ->reply()
                        ->count($replyCount)
                        ->create([
                            'post_id'   => $post->id,
                            'parent_id' => $parentComment->id,
                        ]);
                }
            }

            // প্রথম লুপের ভেতরে ২০% পেন্ডিং কমেন্ট তৈরির লজিক
            $pendingCount = max(1, floor($parentCommentCount * 0.2));

            Comment::factory()
                ->count($pendingCount)
                ->create([
                    'post_id' => $post->id,
                ]);
        }

        // টার্মিনালে সাকসেস মেসেজ প্রিন্ট করা
        $this->command->info('Comment seeded successfully: Parent comments, child replies, and pending comments generated.');
    }
}

```

### সারাংশ ও পরবর্তী পরিকল্পনা

এই পর্বে আমরা বেশ চমৎকার এবং প্রফেশনাল লেভেলের লজিক অ্যাপ্লাই করে `CommentSeeder`-এর কাজ সম্পূর্ণ শেষ করলাম। কোডগুলো একটু জটিল মনে হতে পারে, তাই আপনারা এটি বারবার বাসায় প্র্যাক্টিস করবেন। প্র্যাক্টিস করলে দেখবেন যে জিনিসটা আপনার কাছে একদম সহজ হয়ে যাচ্ছে আশা করি।

আমাদের কমেন্ট সিডারের কাজ এখানেই শেষ। এর পর আমরা যেটা করবো সেটা হচ্ছে আমাদের যে **ডাটাবেজ সিডার (DatabaseSeeder)** আছে, যেটি লারাভেলে ডিফল্ট ভাবে তৈরি করা থাকে, সেটি নিয়ে আমাদের কাজ করতে হবে।

আগামী পর্বে আমরা প্রধান ডাটাবেজ সিডার ফাইলটি সাজিয়ে আমাদের তৈরি করা সমস্ত সিডারগুলোকে একসাথে রান করা শিখবো এবং এর মাধ্যমেই আমাদের ডাটাবেজ রিলেটেড কর্মকাণ্ড সম্পূর্ণ ক্লোজ হয়ে যাবে। পরবর্তী পর্ব থেকে এটি শুরু করতে পারি, আজ এখানেই বিদায় নিচ্ছি!
