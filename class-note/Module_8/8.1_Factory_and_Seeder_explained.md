# ক্লাস নোট: লারাভেল ফ্যাক্টরি এবং সিডার (Laravel Factory & Seeder)

**প্লাটফর্ম:** প্রযুক্তিপ্লাস একাডেমী

## ১. ফ্যাক্টরি (Factory) কী এবং কেন প্রয়োজন?

### ফ্যাক্টরি কী?

ফ্যাক্টরি হলো মডেলের জন্য ডামি বা ফেক ডেটা (Dummy Data) ক্রিয়েট করার একটি **ব্লুপ্রিন্ট (Blueprint)**।

* **অবস্থান:** প্রজেক্টের `database/factories/` ফোল্ডারের ভেতরে সকল ফ্যাক্টরি ফাইল থাকে।
* **মূল কাজ:** টেস্টিং, ফিচার ডেভেলপমেন্ট বা ইউজার ইন্টারফেস (UI) পরীক্ষা করার জন্য যখন বাস্তবসম্মত প্রচুর ডেটার প্রয়োজন হয়, তখন ফ্যাক্টরি আমাদের অটোমেটিক্যালি সেই ডেটা জেনারেট করে দেয়।

> 💡 **নোট:** কেউ যদি অটোমেটিক্যালি ডেটা জেনারেট করতে না চান, তবে মাইগ্রেশন এবং মডেলে কোড করার পর ফ্যাক্টরি বা সিডারের তেমন প্রয়োজন হয় না। সরাসরি কন্ট্রোলার বা রাউটের কাজে চলে যাওয়া যায়।

## ২. ফ্যাক্টরি তৈরি এবং গঠন

### ফ্যাক্টরি তৈরির কমান্ড

ফ্যাক্টরি তৈরির জন্য টার্মিনালে নিচের কমান্ডটি ব্যবহার করা হয়। ক্লাসের নাম লেখার সময় **StudlyCase** (প্রতিটি শব্দের প্রথম অক্ষর বড় হাতের) বজায় রাখতে হয়।

```bash
php artisan make:factory UserFactory --model=User

```

* `--model=User` ফ্ল্যাগটি ব্যবহারের ফলে এই ফ্যাক্টরি ফাইলটি সরাসরি `User` মডেলের সাথে অ্যাটাচ (Attach) হয়ে যায়।

### ফ্যাক্টরি ফাইলের ভেতরের গঠন (Structure)

একটি বেসিক ফ্যাক্টরি ফাইলে প্রধানত `Namespace`, `Model Definition`, এবং `definition()` মেথড থাকে। নিচে একটি উদাহরণ দেওয়া হলো:

```php
namespace Database\Factories;

use App\Models\User;
use Illuminate\Database\Eloquent\Factories\Factory;
use Illuminate\Support\Facades\Hash;

class UserFactory extends Factory
{
    // ফ্যাক্টরি কোন মডেলের সাথে সংযুক্ত তা নির্দেশ করে
    protected $model = User::class;

    /**
     * Define the model's default state.
     */
    public function definition(): array
    {
        return [
            'name' => fake()->name(),
            'email' => fake()->unique()->safeEmail(),
            'password' => Hash::make('password'), // পাসওয়ার্ড হ্যাশ (Hash) করে দেওয়া হচ্ছে
            'role' => fake()->randomElement(['author', 'reader']), // র্যান্ডম রোল সিলেকশন
        ];
    }
}

```

* **fake():** এটি লারাভেলের একটি গ্লোবাল হেলপার, যা মূলত `fakerphp/faker` লাইব্রেরির একটি ইন্সট্যান্স রিটার্ন করে।
* **পাসওয়ার্ড হ্যাশিং:** পাসওয়ার্ড ফিল্ডে সরাসরি স্ট্রিং না রেখে `Hash::make()` ব্যবহার করা হয়, যা পাসওয়ার্ডকে এনকোড করে হিজিবিজি অক্ষরে রূপান্তর করে (নিরাপত্তার স্বার্থে)।

## ৩. ফেকার মেথডসমূহের ব্যবহার (Advanced Faker Methods)

পোস্ট বা অন্যান্য বড় টেবিলের ডেটা জেনারেট করার জন্য আমরা বিভিন্ন ধরনের ফেকার মেথড ব্যবহার করতে পারি। যেমন একটি `PostFactory`-এর উদাহরণ নিচে দেওয়া হলো:

```php
public function definition(): array
{
    return [
        'title' => fake()->sentence(6), // ৬টি শব্দের একটি বাক্য তৈরি করবে
        'slug' => fake()->unique()->slug(5), // ৫ শব্দের ইউনিক ইউআরএল ফ্রেন্ডলি স্লাগ তৈরি করবে
        'excerpt' => fake()->paragraph(2), // ২টি প্যারাগ্রাফ তৈরি করবে
        'content' => fake()->paragraph(5, true), // ৫টি প্যারাগ্রাফের একটি বড় স্ট্রিং তৈরি করবে
        'is_featured' => fake()->boolean(20), // ২০% সম্ভাবনা ট্রু (Yes) এবং ৮০% সম্ভাবনা ফলস (No) হওয়ার
        'views' => fake()->numberBetween(3, 20), // ৩ থেকে ২০ এর মধ্যে যেকোনো র্যান্ডম সংখ্যা
    ];
}

```

## ৪. ফ্যাক্টরি স্টেটস (Factory States)

ফ্যাক্টরি থেকে অতিরিক্ত বা বিশেষ কোনো সুবিধা (যেমন নির্দিষ্ট কোনো রোলের ডেটা) নেওয়ার জন্য আমরা **States** ব্যবহার করি। এটি ফ্যাক্টরি ক্লাসের ভেতরেই মেথড আকারে ডিফাইন করা থাকে।

### স্টেট ডিফাইন করার নিয়ম:

```php
// UserFactory এর ভেতরে স্টেট তৈরি
public function admin(): static
{
    return $this->state(fn (array $attributes) => [
        'role' => 'admin',
    ]);
}

```

### স্টেট ব্যবহার করার নিয়ম:

```php
// এডমিন ইউজার তৈরি করার জন্য
User::factory()->admin()->create();

```

## ৫. ফ্যাক্টরি রিলেশনশিপ (Relationships)

ফ্যাক্টরির মাধ্যমে ডেটা জেনারেট করার সময় মডেলগুলোর মধ্যকার রিলেশনশিপ বা সম্পর্কও সেটআপ করা যায়। এর জন্য লারাভেলে `for()`, `has()`, `recycle()` ইত্যাদি মেথড রয়েছে।

* **নির্দিষ্ট ইউজারের অধীনে পোস্ট তৈরি:**
```php
Post::factory()
    ->count(5)
    ->for(User::factory()->create())
    ->create();

```

## ৬. সিডার (Seeder) কী এবং কেন প্রয়োজন?

### সিডার কী?

সিডার হলো ডেটাবেসে নির্দিষ্ট বা ফিক্সড কিছু ডেটা ইনপুট করার প্রক্রিয়া। প্রজেক্টের `database/seeders/` ফোল্ডারের মধ্যে সকল সিডার ফাইল থাকে।

### ফ্যাক্টরি বনাম সিডার (Factory vs Seeder)

| বৈশিষ্ট্য | ফ্যাক্টরি (Factory) | সিডার (Seeder) |
| --- | --- | --- |
| **ডেটার ধরণ** | ডামি বা ফেক ডেটা (র্যান্ডম)। | নির্দিষ্ট বা ফিক্সড ডেটা (যেমন: সুপার এডমিন অ্যাকাউন্ট ক্রিয়েশন)। |
| **আউটপুট** | প্রতিবার রান করলে ভিন্ন ভিন্ন ডেটা আসে। | প্রতিবার রান করলে একই বা নির্দিষ্ট ডেটা ইনপুট হয়। |

### সিডার ফাইল তৈরির কমান্ড:

```bash
php artisan make:seeder UserSeeder

```

### সিডারের ভেতরের কোড ও কাউন্ট ট্রিকস:

```php
namespace Database\Seeders;

use App\Models\User;
use Illuminate\Database\Seeder;

class UserSeeder extends Seeder
{
    public function run(): void
    {
        // কাউন্ট না দিলে ডিফল্ট ১ জন সুপার এডমিন তৈরি হবে
        User::factory()->state(['role' => 'super_admin'])->create();

        // count(2) দেওয়ার কারণে ২ জন এডমিন তৈরি হবে
        User::factory()->count(2)->admin()->create();

        // ৫ জন অথর তৈরি হবে
        User::factory()->count(5)->state(['role' => 'author'])->create();
    }
}

```

## ৭. ডেটাবেস সিডার এবং নির্ভরতা (DatabaseSeeder & Dependency)

লারাভেলের মূল সিডার ফাইলটি হলো `DatabaseSeeder.php`। প্রজেক্টের সব সিডার ফাইলকে এই ফাইলের `run()` মেথডের ভেতর `call()` মেথডের মাধ্যমে এক্সিকিউট করতে হয়।

### কল করার ক্ষেত্রে ধারাবাহিকতা (Order of Execution)

সিডার কল করার সময় খেয়াল রাখতে হবে—**যে মডেলটি অন্য মডেলের ওপর নির্ভরশীল, তাকে নিচে রাখতে হবে। আর যার ওপর নির্ভরশীল, তাকে উপরে রাখতে হবে।**

```php
public function run(): void
{
    $this->call([
        UserSeeder::class,     // ইউজার আগে তৈরি হতে হবে
        CategorySeeder::class, // ক্যাটাগরি তৈরি হবে
        TagSeeder::class,      // ট্যাগ তৈরি হবে
        PostSeeder::class,     // ইউজার ও ক্যাটাগরি না থাকলে পোস্ট তৈরি হতে পারবে না
        CommentSeeder::class,  // পোস্ট না থাকলে কমেন্ট তৈরি হতে পারবে না
    ]);
}

```

## ৮. প্রয়োজনীয় লারাভেল কারিগরী কমান্ডসমূহ (Artisan Commands)

ডেটা সেটিং ও মাইগ্রেশনের জন্য নিচে বহুল ব্যবহৃত কমান্ডগুলোর কার্যকারিতা দেওয়া হলো:

| কমান্ড | কাজ |
| --- | --- |
| `php artisan db:seed` | `DatabaseSeeder`-এ থাকা সকল সিডার ফাইল একসাথে রান করে। |
| `php artisan db:seed --class=UserSeeder` | নির্দিষ্ট করে শুধু `UserSeeder` ফাইলটি রান করার জন্য। |
| `php artisan migrate:fresh --seed` | **(সবচেয়ে বেশি ব্যবহৃত)** সমস্ত টেবিল ড্রপ (ডিলিট) করে, নতুন করে মাইগ্রেশন রান করে টেবিল তৈরি করে এবং একসাথে সিডিং সম্পন্ন করে। |
| `php artisan migrate:refresh --seed` | মাইগ্রেশনের `down()` মেথড কল করে রোলব্যাক করে, তারপর আবার রিফ্রেশ করে সিডিং রান করে। (তুলনামূলক কম ব্যবহৃত)। |
| `php artisan migrate --seed` | নতুন মাইগ্রেশন ফাইলের ক্ষেত্রে টেবিল তৈরি করে এবং সরাসরি সিড রান করে। |
