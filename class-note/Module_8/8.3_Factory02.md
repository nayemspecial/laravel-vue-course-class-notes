# ক্লাস নোট: লারাভেল ফ্যাক্টরি এবং সিডার (Laravel Factory & Seeder)

**বিষয়:** ৮.৩ ফ্যাক্টরি ফাইলের ভেতরে বিভিন্ন ইউজার রোলের জন্য কাস্টম স্টেট তৈরি করা, রিটার্ন টাইপ এবং টাইপ হিন্টিং

---

## ১. ফ্যাক্টরি স্টেট (Factory State) এবং ফিক্সড ইনফো সেটআপ

* ডাটাবেজে যখন আমরা ডামি ডাটা জেনারেট করি, তখন কিছু নির্দিষ্ট ইউজার রোলের জন্য আমাদের ফিক্সড ইনফো বা নির্দিষ্ট ডেটার প্রয়োজন হয় (যেমন: সুপার এডমিন)।
* এই ফিক্সড ইনফোগুলো যেন ফেকার (Faker) এর রেন্ডম ডাটা থেকে না আসে এবং যতবারই আমরা ডাটা রিফ্রেশ বা চেঞ্জ করব, এই নির্দিষ্ট ডাটাগুলো যেন ফিক্সড থাকে, সেই সুবিধা রাখার জন্যই আমরা স্টেট ব্যবহার করব।
* এটি যে শুধুমাত্র সুপার এডমিনের জন্য করা যাবে ব্যাপারটা তা নয়, যেকোনো নামেই এই ফ্যাসিলিটিসটা আমরা প্রজেক্টের প্রয়োজন অনুযায়ী সেটআপ করতে পারব।

## ২. লারাভেলে রিটার্ন টাইপ এবং টাইপ হিন্টিং (Type Hinting)

* লারাভেল ১১ বা আধুনিক সংস্করণে আমরা মেথডের শেষে ক্লোন (`:`) ব্যবহার করে রিটার্ন টাইপ (যেমন: `: static`) লিখে দিতে পারি, যা স্টেট নির্দিষ্ট ইউজার তৈরিতে সাহায্য করে।
* এছাড়াও কলব্যাক ফাংশনে প্যারামিটারের আগে টাইপ বলে দেওয়াকে **টাইপ হিন্টিং (Type Hinting)** বলে (যেমন: `array $attributes`)।

## ৩. বিভিন্ন রোলের জন্য কাস্টম স্টেট মেথডসমূহ

### ক) সুপার এডমিন সেটআপ (`superAdmin`)

সুপার এডমিনের ক্ষেত্রে নাম, ইমেইল এবং রোল সম্পূর্ণ ফিক্সড থাকবে। এখানে একটি বিজনেস ইমেইল ব্যবহার করা হয়েছে যা পরবর্তীতে লগইন বা অন্যান্য টেস্টের কাজে লাগবে। এছাড়া সুপার এডমিন সবসময় ফিচারড (`'is_featured' => true`) থাকবে।

```php
public function superAdmin(): static
{
    return $this->state(fn (array $attributes) => [
        'name' => 'সুপার এডমিন',
        'email' => 'admin@classicd.com', // বিজনেস ইমেইল
        'role' => 'super_admin',
        'is_featured' => true,
    ]);
}

```

### খ) এডমিন সেটআপ (`admin`)

এডমিনের ক্ষেত্রে নাম এবং ইমেইল ফিক্সড করার প্রয়োজন নেই, সেগুলো ফেকার লাইব্রেরি থেকে অটোমেটিক জেনারেট হবে। তবে রোল ফিক্সড থাকবে এবং `is_featured` এর ক্ষেত্রে ৫০% ট্রু বা ফলস হওয়ার একটি সম্ভাবনা বা রেট সেট করে দেওয়া হয়েছে।

* **লারাভেলে `fake()` এবং `faker` এর পার্থক্য:** লারাভেলে `fake()` হচ্ছে একটি মেথড বা হেলপার ফাংশন (যেখানে ব্র্যাকেট ব্যবহার করতে হয়), আর `faker` হলো প্রপার্টি। বুলিয়ানের ক্ষেত্রে আমরা `fake()->boolean(50)` ব্যবহার করে অর্ধেক ফিচারড এবং অর্ধেক নন-ফিচারড করতে পারি।

```php
public function admin(): static
{
    return $this->state(fn (array $attributes) => [
        'role' => 'admin',
        'is_featured' => fake()->boolean(50), // ৫০% সম্ভাবনা
    ]);
}

```

### গ) অথর সেটআপ (`author`)

অথরের জন্য রোল হবে `author` এবং তাদের ফিচারড হওয়ার সম্ভাবনা ৩০% বা তার কাছাকাছি রাখা হয়েছে।

```php
public function author(): static
{
    return $this->state(fn (array $attributes) => [
        'role' => 'author',
        'is_featured' => fake()->boolean(30),
    ]);
}

```

### ঘ) রিডার সেটআপ (`reader`)

রিডারদের ক্ষেত্রে রোল হবে `reader` এবং তারা যেহেতু শুধু কনটেন্ট পড়বে, তাই তাদের আমরা ফিচারড করব না বা চাইল্ডে নিব না।

```php
public function reader(): static
{
    return $this->state(fn (array $attributes) => [
        'role' => 'reader',
        'is_featured' => false,
    ]);
}

```

## ৪. সম্পূর্ণ কোড স্ট্রাকচার (`UserFactory.php`)

উপরে আলোচিত কাস্টম স্টেট মেথডগুলোর সম্পূর্ণ কোড ব্লকের রূপ নিচে দেওয়া হলো। কলব্যাক ফাংশন এবং অ্যারো ফাংশন (`fn () =>`) এর এই ব্যবহারগুলো বেশি বেশি প্র্যাকটিস করতে হবে।

```php
<?php

namespace Database\Factories;

use App\Models\User;
use Illuminate\Database\Eloquent\Factories\Factory;
use Illuminate\Support\Facades\Hash;
use Illuminate\Support\Str;

class UserFactory extends Factory
{
    // definition() মেথডটি পূর্বে ডিফাইন করা আছে...

    /**
     * Indicate that the user is a super admin.
     */
    public function superAdmin(): static
    {
        return $this->state(fn (array $attributes) => [
            'name' => 'সুপার এডমিন',
            'email' => 'admin@classicd.com',
            'role' => 'super_admin',
            'is_featured' => true,
        ]);
    }

    /**
     * Indicate that the user is an admin.
     */
    public function admin(): static
    {
        return $this->state(fn (array $attributes) => [
            'role' => 'admin',
            'is_featured' => fake()->boolean(50),
        ]);
    }

    /**
     * Indicate that the user is an author.
     */
    public function author(): static
    {
        return $this->state(fn (array $attributes) => [
            'role' => 'author',
            'is_featured' => fake()->boolean(30),
        ]);
    }

    /**
     * Indicate that the user is a reader.
     */
    public function reader(): static
    {
        return $this->state(fn (array $attributes) => [
            'role' => 'reader',
            'is_featured' => false,
        ]);
    }
}

```

*পরবর্তীতে যখন আমরা এই ফ্যাক্টরি ব্যবহার করে ডাটা জেনারেট করব, তখন কোনো কিছু বাদ থাকলে বা এরর আসলে আমরা সেগুলোর সলুশন করার চেষ্টা করব। আজকের পর্ব এখানেই শেষ, আগামী পর্বে আমরা অন্য ভিডিওতে বাকি কাজ শুরু করব।*
