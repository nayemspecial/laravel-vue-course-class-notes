# ক্লাস নোট: লারাভেল ফ্যাক্টরি এবং সিডার (Laravel Factory & Seeder)

**প্লাটফর্ম:** প্রযুক্তিপ্লাস একাডেমী (ProjuktiPlus Academy)

**বিষয়:** ৮.৬ পোস্ট ফ্যাক্টরি তৈরি এবং মডেল এলাইনমেন্ট, প্রয়োজনীয় ক্লাস ইমপোর্ট, সম্পূর্ণ পোস্ট ফ্যাক্টরি কোড   স্ট্রাকচার ও বিস্তারিত আলোচনা। 

প্রযুক্তি প্লাস একাডেমির পক্ষ থেকে আপনাদের সবাইকে স্বাগতম জানাচ্ছি। এই পর্বে আমরা আমাদের পোস্ট ফ্যাক্টরি নিয়ে কাজ শুরু করতে চাই। আমাদের পোস্ট টেবিলের মধ্যে মিনিমাম ১২ টার মতো ফিল্ড ছিল, তাই এখানে আমাদের ভালোই একটা সময় লাগবে। প্রতিটি ফিল্ড নিখুঁতভাবে এবং লজিক্যালি ফেকার ডেটা দিয়ে কীভাবে সেটআপ করবেন, তা নিচে বিস্তারিত আলোচনা ও কোডসহ দেওয়া হলো।

---

### ১. পোস্ট ফ্যাক্টরি তৈরি এবং মডেল এলাইনমেন্ট

প্রথমে আমরা আমাদের কোড এডিটরে যাব এবং টার্মিনালটি ওপেন করে পোস্ট ফ্যাক্টরিটা ক্রিয়েট করব। ফ্যাক্টরি তৈরি করার সময় আমরা এটিকে সরাসরি `Post` মডেলের সাথে এলাইন বা কানেক্ট করে দেব।

**টার্মিনাল কমান্ড:**

```bash
php artisan make:factory PostFactory --model=Post

```

এই কমান্ডটি রান করার পর আমাদের `database/factories` ডিরেক্টরিতে `PostFactory.php` ফাইলটি ক্রিয়েট হয়ে যাবে।

### ২. প্রয়োজনীয় ক্লাস ইমপোর্ট (Use Classes)

পোস্ট ফ্যাক্টরিতে আমাদের বেশ কিছু ক্লাস ব্যবহার করতে হবে। যেহেতু পোস্টের সাথে ইউজার এবং ক্যাটাগরির রিলেশনশিপ আছে, তাই আমাদের এই মডেল বা ক্লাসগুলো লাগবে। কোড এডিটরে শর্টকাট হিসেবে `Alt + Shift + Down Arrow` ইউজ করে আমরা লাইন ডুপ্লিকেট করে ক্লাসগুলো এড করে নিতে পারি।

```php
use App\Models\Post;
use App\Models\User;
use App\Models\Category;
use Illuminate\Database\Eloquent\Factories\Factory;
use Illuminate\Support\Str;

```

### ৩. সম্পূর্ণ পোস্ট ফ্যাক্টরি কোড স্ট্রাকচার (Complete Post Factory Code)

নিচে আলোচনার ওপর ভিত্তি করে প্রতিটি ফিল্ডের লজিকসহ সম্পূর্ণ `PostFactory` কোডটি দেওয়া হলো:

```php
<?php

namespace Database\Factories;

use App\Models\Post;
use App\Models\User;
use App\Models\Category;
use Illuminate\Database\Eloquent\Factories\Factory;
use Illuminate\Support\Str;

class PostFactory extends Factory
{
    /**
     * The name of the factory's corresponding model.
     *
     * @var string
     */
    protected $model = Post::class;

    /**
     * Define the model's default state.
     *
     * @return array<string, mixed>
     */
    public function definition(): array
    {
        // বাংলাদেশি ডাটা জেনারেট করার জন্য ফেকার ইনস্ট্যান্স
        $faker = \Faker\Factory::create('bn_BD');

        // ১. টাইটেল ভেরিয়েবল (ইউনিক এবং ৬ শব্দের সেন্টেন্স)
        $title = $this->faker->unique()->sentence(6, true);

        // ২. স্লাগ ফরমেটিং
        $slug = Str::slug($title);

        // ৩. পাবলিশড অ্যাট (পাবলিশ টাইম) ভেরিয়েবল সেটআপ
        // গত ১ বছরের মধ্যে রেন্ডম ডেট এবং ৯০% ক্ষেত্রে ডেটা থাকবে, ১০% ক্ষেত্রে নাল হবে
        $publishedAt = $this->faker->optional(0.9)->dateTimeBetween('-1 year', 'now');

        return [
            'title'             => $title,
            'slug'              => $slug,
            
            // ৪. এক্সাপ্ট বা শর্ট কন্টেন্ট (১৫ শব্দের সেন্টেন্স)
            'excerpt'           => $this->faker->sentence(15, true),
            
            // ৫. মেইন কন্টেন্ট (১০টি প্যারাগ্রাফ বিশিষ্ট বড় টেক্সট)
            'content'           => $this->faker->paragraphs(10, true),
            
            // ৬. ফিচারড ইমেজ (Unsplash থেকে রেন্ডম এবং ইউনিক আইডি দিয়ে ইমেজ ইউআরএল জেনারেট)
            'featured_image'    => 'https://images.unsplash.com/photo-' . $this->faker->numberBetween(1000000000000, 1999999999999) . '?w=800&auto=format&fit=crop&q=60',
            
            // ৭. ইউজার আইডি (ডাটাবেজে অথর রোল ওয়ালা ইউজার থাকলে রেন্ডমলি নিবে, না থাকলে নতুন অথর তৈরি করবে)
            'user_id'           => User::where('role', 'author')->inRandomOrder()->first()?->id ?? User::factory()->state(['role' => 'author'])->create()->id,
            
            // ৮. ক্যাটাগরি আইডি (ডাটাবেজে ক্যাটাগরি থাকলে রেন্ডমলি নিবে, না থাকলে নতুন ক্রিয়েট করবে)
            'category_id'       => Category::inRandomOrder()->first()?->id ?? Category::factory()->create()->id,
            
            // ৯. স্ট্যাটাস (টার্নারি অপারেটর এবং published_at এর ওপর ভিত্তি করে)
            'status'            => $publishedAt ? 'published' : 'draft',
            
            // ১০. ইজ ফিচারড (১০% ক্ষেত্রে ট্রু বা ১ হবে, ৯০% ক্ষেত্রে ফলস বা ০)
            'is_featured'       => $this->faker->boolean(10),
            
            // ১১. ইজ এডিটর চয়েস (৫% ক্ষেত্রে ট্রু বা ১ হবে)
            'is_editor_choice'  => $this->faker->boolean(5),
            
            // ১২. রিডিং টাইম (৫ থেকে ২৫ মিনিটের মধ্যে রেন্ডম নাম্বার)
            'reading_time'      => $this->faker->numberBetween(5, 25),
            
            // পাবলিশ টাইম অ্যাসাইন
            'published_at'      => $publishedAt,
        ];
    }
}

```

### ৪. প্রতিটি ফিল্ডের বিস্তারিত লজিক ও ব্যাখ্যা

* **বাংলাদেশি ডাটা ও ফেকার:** ক্যাটাগরি ফ্যাক্টরির মতো এখানেও আমরা শুরুতে `\Faker\Factory::create('bn_BD')` ইনস্ট্যান্সটি ব্যবহার করেছি যাতে লোকাল ডেটা সাপোর্ট করে।
* **Title:** টাইটেলটিকে আমরা ভেরিয়েবলে নিয়েছি কারণ এটি স্লাগেও লাগবে। ডেটাবেজে সেভ করার সময় যেন ডুপ্লিকেট হয়ে ঝামেলা না করে, তাই সতর্কতা মূলক `unique()` ব্যবহার করেছি এবং ৬ শব্দের সেন্টেন্স হিসেবে `sentence(6, true)` দিয়েছি।
* **Slug:** `Illuminate\Support\Str` ক্লাসের স্লাগ মেথড বা `Str::slug($title)` দিয়ে সরাসরি ইউআরএল ফ্রেন্ডলি স্লাগ তৈরি করা হয়েছে।
* **Excerpt:** এটি মূলত হোম পেজে পোস্ট ডিটেইলসে যাওয়ার আগে যে ১৫ ওয়ার্ডের মতো শর্ট টেক্সট থাকে, সেটির জন্য `sentence(15, true)` ব্যবহার করা হয়েছে।
* **Content:** পোস্টের মূল কন্টেন্ট অনেক বড় হয়। তাই এখানে ১০টি রেন্ডম প্যারাগ্রাফ জেনারেট করার জন্য `paragraphs(10, true)` নেওয়া হয়েছে।
* **Featured Image (Unsplash URL Logic):** রেন্ডম ইমেজের জন্য আমরা অনলাইন ডামি ইমেজ বা Unsplash-এর রিয়েল ইমেজ ফরম্যাট ব্যবহার করেছি। ব্রাউজারে Unsplash-এর ইমেজ ওপেন করলে দেখা যায় ইউআরএলটি `images.unsplash.com/photo-` এর পর একটি নির্দিষ্ট আইডি এবং শেষে কিছু কোয়েরি প্যারামিটার থাকে। আমরা `numberBetween()` দিয়ে সেই আইডি পার্টটুকুকে রেন্ডমাইজ করেছি যাতে প্রতিবার আলাদা ছবি আসে।
* **User ID (নাল কোয়ালিসিং অপারেটর):** ডাটাবেজে যদি আগে থেকে কোনো 'author' রোলের ইউজার থাকে, তবে `inRandomOrder()->first()` এর মাধ্যমে রেন্ডমলি সেটি নিয়ে আসবে। আর যদি না থাকে, তবে নালিশ কোয়ালিসিং বা নাল কোয়ালিসিং অপারেটর (`??`) এর মাধ্যমে নতুন একটি ইউজার ফ্যাক্টরি রান করে অথর ক্রিয়েট করে তার আইডি বসিয়ে দেবে। এর ফলে এরর আসার সম্ভাবনা কম থাকে।
* **Category ID:** ইউজারের মতোই ক্যাটাগরি থাকলে ডাটাবেজ থেকে রেন্ডমলি আইডি পিক করবে, আর না থাকলে নতুন ক্যাটাগরি ইনস্ট্যান্টলি ক্রিয়েট করে নিবে।
* **Published At (ভবিষ্যতের টাইম এভয়েড করা):** ডামি ডাটা জেনারেট করার সময় যদি ভবিষ্যতের (যেমন ২০২৭ সালের) কোনো ডেট চলে আসে তবে সেটি লজিক্যাল হবে না। তাই আমরা লাস্ট ইয়ার বা গত এক বছরের রেন্ডম ডেট ও টাইম আনার জন্য `dateTimeBetween('-1 year', 'now')` ব্যবহার করেছি। এখানে `optional(0.9)` ব্যবহার করে বোঝানো হয়েছে ৯০% ক্ষেত্রে ডেট থাকবে এবং ১০% ক্ষেত্রে ডেটা থাকবে না (অর্থাৎ নাল হবে)।
* **Status:** এখানে আমরা টার্নারি অপারেটর ইউজ করেছি। যদি উপরে ডিফাইন করা `$publishedAt` সত্য হয় (অর্থাৎ ডেট থাকে), তাহলে স্ট্যাটাস হবে `'published'`, আর যদি সত্য না হয় বা নাল হয়, তবে স্ট্যাটাস হবে `'draft'`.
* **Is Featured & Is Editor Choice:** এগুলো মূলত বুলিয়ান ফিল্ড। সব পোস্ট তো ফিচারড বা এডিটর চয়েস হয় না। তাই পারসেন্টেজ ঠিক করে দিয়েছি; ১০% পোস্ট ফিচারড হবে `boolean(10)` এবং ৫% পোস্ট এডিটর চয়েস হবে `boolean(5)`.
* **Reading Time:** একটি পোস্ট পড়তে কত সময় লাগবে তার জন্য `numberBetween(5, 25)` দিয়ে ৫ থেকে ২৫ এর মধ্যে রেন্ডম মিনিট সেটআপ করা হয়েছে।

### সারসংক্ষেপ:

পোস্ট ফ্যাক্টরির বেসিক ব্লুপ্রিন্ট ও ফিল্ড সেটআপ আমরা চমৎকারভাবে শেষ করলাম। তবে পোস্ট ফ্যাক্টরিতে আমাদের আরও অনেক কাজ আছে। যেমন কাস্টিং বা স্টেটের জন্য নির্দিষ্ট পাবলিক ফাংশন তৈরি করা (যেমন ইউজার ফ্যাক্টরিতে রিডার, অথর বা এডমিনের জন্য আলাদা স্টেট করা হয়েছিল, তেমনি এখানেও ইজ ফিচারড বা পাবলিশের জন্য স্টেট লজিক লিখতে হবে)। সেই লজিক্যাল ব্যাপার স্যাপারগুলো আমরা আমাদের পরবর্তী পর্বে বা আগামী পর্বে নতুন একটি ভিডিওতে শিখবো। এই পর্বটি আমরা এখানেই শেষ করছি।
