# ক্লাস নোট: লারাভেল ফ্যাক্টরি এবং সিডার (Laravel Factory & Seeder)

**প্লাটফর্ম:** প্রজু্ক্তিপ্লাস একাডেমী (ProjuktiPlus Academy)

**বিষয়:** ৮.৭ পোস্ট ফ্যাক্টরির মধ্যে কিছু কাস্টম স্টেট তৈরি,এরো ফাংশন (fn) ব্যবহার করে state মেথড কল করা, প্যারামিটার হিসেবে array $attributes কে টাইপ হিন্টিং করা। 

প্রযুক্তি প্লাস একাডেমির পক্ষ থেকে সবাইকে স্বাগতম। গত পর্বে আমরা আমাদের `PostFactory`-র বেসিক ব্লুপ্রিন্ট ও ডেফিনেশন অ্যারে তৈরি করা শিখেছি। এই পর্বে আমরা আমাদের পোস্ট ফ্যাক্টরির মধ্যে বেশ কিছু কাস্টম স্টেট (State) মেথড লিখবো।

ফ্যাক্টরি স্টেট মূলত আমাদের ডামি ডেটা জেনারেট করার সময় নির্দিষ্ট কিছু কন্ডিশন বা লজিক জোরপূর্বক (forcefully) সেট করতে সাহায্য করে। যেহেতু আমরা লারাভেলের লেটেস্ট ভার্সন (Laravel 13) ব্যবহার করছি, তাই আমরা এখানে প্রপার রিটার্ন টাইপ এবং মডার্ন সিনট্যাক্স ব্যবহার করে কোডগুলো লিখবো।

### ১. `published()` স্টেট মেথড তৈরি

প্রথমে আমরা কাজ করবো পাবলিশড পোস্ট নিয়ে। আমরা চাই এমন একটা স্টেট তৈরি করতে, যা কল করলে পোস্টের স্ট্যাটাস অবশ্যই `published` হবে এবং একটি রিয়েল পাবলিশ ডেট থাকবে (কোনো নাল ভ্যালু থাকবে না)।

* **মেথড নেম:** `published`
* **রিটার্ন টাইপ:** `static` (লারাভেলের লেটেস্ট স্ট্যান্ডার্ড অনুযায়ী)
* **লজিক:** এখানে আমরা এরো ফাংশন (`fn`) ব্যবহার করে `state` মেথড কল করবো। প্যারামিটার হিসেবে `array $attributes` কে টাইপ হিন্টিং করে দিতে পারি। অ্যাসোসিয়েটিভ এরের ভেতরে `status` হবে `'published'` এবং `published_at` এর জন্য গত ১ বছরের মধ্যে যেকোনো একটি রেন্ডম ডেট ও টাইম জেনারেট করবো (এখানে কোনো `optional()` ব্যবহার করা যাবে না, অর্থাৎ ডেট থাকতেই হবে)।

```php
    /**
     * Indicate that the post is published.
     */
    public function published(): static
    {
        return $this->state(fn (array $attributes) => [
            'status' => 'published',
            'published_at' => fake()->dateTimeBetween('-1 year', 'now'),
        ]);
    }

```

### ২. `featured()` স্টেট মেথড তৈরি

এবার আমরা তৈরি করবো `featured` স্টেট মেথড। যদি আমরা কোনো পোস্টকে জোর করে ফিচারড পোস্ট বানাতে চাই, তবে তার সাথে কিছু লজিক জড়িয়ে থাকে। লজিকটা হলো—একটি পোস্ট যদি ফিচারড হয়, তবে সেটি অবশ্যই ডাটাবেজে `published` অবস্থায় থাকতে হবে এবং তার একটি নির্দিষ্ট পাবলিশ টাইমও থাকতে হবে। তাই এই মেথডে আমরা তিনটি ফিল্ড একসাথে সেটআপ করবো।

* **মেথড নেম:** `featured`
* **লজিক:** কোড এডিটরে আগের মেথডটি ডুপ্লিকেট করে সহজেই এটি তৈরি করতে পারেন। এখানে `is_featured` ফিল্ডের ভ্যালু জোর করে `true` করে দেওয়া হচ্ছে, পাশাপাশি `status` হবে `'published'` এবং `published_at` এ রেন্ডম ডেটটাইম বসে যাবে। সবকিছুই এখানে লজিক্যাল।

```php
    /**
     * Indicate that the post is featured.
     */
    public function featured(): static
    {
        return $this->state(fn (array $attributes) => [
            'is_featured' => true,
            'status' => 'published',
            'published_at' => fake()->dateTimeBetween('-1 year', 'now'),
        ]);
    }

```

### ৩. `editorsChoice()` স্টেট মেথড তৈরি

ফিচারড মেথডের মতোই আমাদের আরেকটি ফিল্ড ছিল `is_editor_choice`। কোনো পোস্টকে এডিটরস চয়েস হিসেবে মার্ক করতে হলেও পোস্টটি অবশ্যই পাবলিশড হতে হবে এবং তার একটি পাবলিশ ডেট থাকতে হবে।

* **মেথড নেম:** `editorsChoice`
* **লজিক:** আগের মেথডটি নিচে পেস্ট করে আমরা একটু এডিট করে নেবো। এখানে আমাদের `is_editor_choice` ফিল্ডের ভ্যালু আমরা করে দেবো `true`। এর ফলে আমাদের এডিটরের চয়েসটা জোর করে ট্রু হয়ে যাবে এবং বাকি পাবলিশড লজিকগুলোও ঠিকঠাক কাজ করবে।

```php
    /**
     * Indicate that the post is an editor's choice.
     */
    public function editorsChoice(): static
    {
        return $this->state(fn (array $attributes) => [
            'is_editor_choice' => true,
            'status' => 'published',
            'published_at' => fake()->dateTimeBetween('-1 year', 'now'),
        ]);
    }

```

### ৪. `draft()` স্টেট মেথড তৈরি

আমাদের স্ট্যাটাসের মধ্যে যেমন `published` আছে, তেমনই আরেকটি গুরুত্বপূর্ণ অপশন হলো `draft`। যদি কোনো পোস্ট ড্রাফট বা খসড়া হিসেবে সেভ হয়, তবে লজিক্যালি তার কোনো পাবলিশড ডেট থাকতে পারবে না। অর্থাৎ, তার `published_at` ফিল্ডটি অবশ্যই `null` হতে হবে।

* **মেথড নেম:** `draft`
* **লজিক:** এই মেথডে আমরা `status` ফিল্ডের ভ্যালু সেট করবো `'draft'` এবং `published_at` ফিল্ডের ভ্যালু সরাসরি করে দেবো `null`। ড্রাফট পোস্টের ক্ষেত্রে এই লজিকটি মেইনটেইন করা অত্যন্ত জরুরী।

```php
    /**
     * Indicate that the post is a draft.
     */
    public function draft(): static
    {
        return $this->state(fn (array $attributes) => [
            'status' => 'draft',
            'published_at' => null,
        ]);
    }

```

### ৫. সম্পূর্ণ পোস্ট ফ্যাক্টরি স্টেট কোড স্ট্রাকচার

আমাদের ডুপ্লিকেট করা এবং নতুন তৈরি করা মেথডগুলো একসাথে `PostFactory` ক্লাসের নিচে যেভাবে বসে যাবে, তার সম্পূর্ণ কোড রূপটি নিচে দেওয়া হলো। প্রতিটি লাইনের শেষে সেমিকোলন (`;`) ঠিকঠাক দেওয়া হয়েছে কিনা তা অবশ্যই খেয়াল রাখবেন।

```php
<?php

namespace Database\Factories;

use App\Models\Post;
use Illuminate\Database\Eloquent\Factories\Factory;

class PostFactory extends Factory
{
    protected $model = Post::class;

    public function definition(): array
    {
        // পূর্বে তৈরি করা ব্লুপ্রিন্ট ডেফিনেশন এখানে থাকবে
        return [
            // ...
        ];
    }

    /**
     * Indicate that the post is published.
     */
    public function published(): static
    {
        return $this->state(fn (array $attributes) => [
            'status' => 'published',
            'published_at' => fake()->dateTimeBetween('-1 year', 'now'),
        ]);
    }

    /**
     * Indicate that the post is featured.
     */
    public function featured(): static
    {
        return $this->state(fn (array $attributes) => [
            'is_featured' => true,
            'status' => 'published',
            'published_at' => fake()->dateTimeBetween('-1 year', 'now'),
        ]);
    }

    /**
     * Indicate that the post is an editor's choice.
     */
    public function editorsChoice(): static
    {
        return $this->state(fn (array $attributes) => [
            'is_editor_choice' => true,
            'status' => 'published',
            'published_at' => fake()->dateTimeBetween('-1 year', 'now'),
        ]);
    }

    /**
     * Indicate that the post is a draft.
     */
    public function draft(): static
    {
        return $this->state(fn (array $attributes) => [
            'status' => 'draft',
            'published_at' => null,
        ]);
    }
}

```

### ক্লাস রিভিউ ও আগামী পর্বের পরিকল্পনা:

এই পর্বে আমরা আমাদের `PostFactory`-র ভেতরের সমস্ত গুরুত্বপূর্ণ মেথড ও স্টেট লজিকগুলো অত্যন্ত নিখুঁত ও লজিক্যাল উপায়ে সাজিয়ে ফেললাম। এই মুহূর্তে আমাদের পোস্ট ফ্যাক্টরিতে আর নতুন কিছু প্রয়োজন বলে মনে হচ্ছে না। পরবর্তীতে যদি প্রজেক্টের প্রয়োজনে কোনো স্টেট এড করতে হয়, তা আমরা পরে এড করে নিতে পারবো।

আপনারা এই লজিকগুলো নিয়ে আরেকটু বেশি পড়াশোনা ও প্র্যাকটিস করবেন, তাহলে এই স্টেট মেথডগুলো আপনাদের কাছে একদম সহজ হয়ে যাবে। আমাদের পোস্ট ফ্যাক্টরির কাজ এখানেই শেষ। এর পরের পর্বে আমরা আমাদের কমেন্ট ফ্যাক্টরি (`CommentFactory`) ক্রিয়েট করবো এবং তারপর আমরা সরাসরি ডাটাবেজ সিডারে (`DatabaseSeeder`) চলে যাবো, যেখানে এই ফ্যাক্টরিগুলোকে একসাথে কল করে হাজার হাজার ডামী ডেটা এক ক্লিকে জেনারেট করা শিখবো।

আজকে এই পর্যন্তই। আগামী ক্লাসে সবার সাথে আবার দেখা হচ্ছে। ধন্যবাদ!
