# লারাভেল কমেন্ট ফ্যাক্টরি (Comment Factory) সেটআপ ও ডেফিনেশন তৈরি

**প্লাটফর্ম:** প্রজু্ক্তিপ্লাস একাডেমী (ProjuktiPlus Academy)

**বিষয়:** ৮.৮ কমেন্ট ফ্যাক্টরি ফাইল ক্রিয়েট করা, প্রয়োজনীয় মডেল ও নেমস্পেস ইম্পোর্ট, নির্দিষ্ট বাংলা কমেন্ট অ্যারে ইউজ করা, কমেন্ট মাইগ্রেশন অনুযায়ী রিটার্ন অ্যারে। 

আজকে আমরা আমাদের প্রজেক্টের **কমেন্ট ফ্যাক্টরি (Comment Factory)** নিয়ে কাজ শুরু করবো। কমেন্ট ফ্যাক্টরি তৈরির মাধ্যমে আমরা আমাদের পোস্টগুলোর জন্য ডায়নামিক এবং লজিক্যাল ডামি কমেন্ট জেনারেট করা শিখবো। নিচে পুরো প্রসেসটি বিস্তারিত ক্লাস নোট আকারে ধাপে ধাপে দেওয়া হলো:

---

### ১. কমেন্ট ফ্যাক্টরি ফাইল ক্রিয়েট করা

কমেন্ট ফ্যাক্টরি নিয়ে কাজ শুরু করার ক্ষেত্রে আমাদের প্রথমে ফাইলটা ক্রিয়েট করে নিতে হবে। তার জন্য আমরা টার্মিনালে আর্টিজান কমান্ড ব্যবহার করবো। কমান্ডটি লেখার সময় খেয়াল রাখবেন `make:factory`-তে যেন বানানে ভুল না হয় (অনেকে 'n' বা 'm' নিয়ে কনফিউজড হন, এটি হবে `make`)।

আমরা কমেন্ট মডেলের সাথে এটিকে এটাচ বা লিঙ্ক করে দেওয়ার জন্য `--model` ফ্ল্যাগ ব্যবহার করবো:

```bash
php artisan make:factory CommentFactory --model=Comment

```

এই কমান্ডটি রান করলে আমাদের ডাটাবেজ ফ্যাক্টরি ডিরেক্টরিতে `CommentFactory.php` নামে একটি ফাইল সাকসেসফুলি ক্রিয়েট হবে।

---

### ২. প্রয়োজনীয় মডেল ও নেমস্পেস ইম্পোর্ট (Add Classes)

কমেন্ট ফ্যাক্টরির ভেতরে আমাদের বেশ কিছু কোড লিখতে হবে এবং অন্যান্য মডেলের ক্লাসগুলোকে অ্যাড করতে হবে। কমেন্টের সাথে যেহেতু **Post** এবং **User** মডেলের রিলেশনশিপ আছে, তাই কমেন্ট জেনারেট করার সময় এই দুইটা মডেলকেও আমাদের এখানে লাগবে।

ফাইলের একদম উপরে আমরা এই ক্লাসগুলোকে ইম্পোর্ট করে নেবো:

```php
use App\Models\Comment;
use App\Models\Post;
use App\Models\User;

```

---

### ৩. কন্টেন্টের জন্য কাস্টম বাংলা কমেন্ট অ্যারে (Fixed Data Setup)

আমরা যদি চাই যে আমাদের কমেন্টগুলো একদম রেন্ডম প্যারাগ্রাফ না হয়ে কিছুটা নির্দিষ্ট বা রিয়েলিস্টিক হোক, তাহলে আমরা নিজেদের মতো ২০টি বা ৩০টি কমেন্ট ফিক্সড করে দিতে পারি। আমরা আমাদের `definition()` মেথডের ভেতরে ল্যাঙ্গুয়েজ লোকালের নিচে `$comments` নামে একটি ভেরিয়েবল নেবো এবং তার ভেতরে সিঙ্গেল কোটেশন ব্যবহার করে কিছু রিয়েলিস্টিক বাংলা কমেন্ট অ্যাসাইন করবো:

```php
// ল্যাঙ্গুয়েজের জন্য লোকাল সেটআপ (প্রয়োজনে)
// fake()->locale('bn_BD'); 

$comments = [
    'অসাধারণ! এই অসাধারণ আর্টিকেল।',
    'খুবই উপকারী কন্টেন্ট।',
    'ধন্যবাদ, এত সুন্দর করে বুঝিয়েছেন।',
];

```

*নোট: আমরা ইউজারেও কিন্তু এভাবে নির্দিষ্ট ডেটা ফিক্সড করে এসেছিলাম (যেমন: সুপার এডমিন রোল)। এখানে মাত্র কয়েকটি কমেন্ট ফিক্সড করে আমরা রিটার্ন অ্যারেতে চলে যাবো। যদি আমরা এই বিশাল অ্যারে সরাসরি রিটার্নের ভেতর লিখতাম তবে কোড দেখতে সুন্দর লাগতো না, তাই এটিকে উপরে ডিফাইন করে নেওয়া হয়েছে।*

### ৪. কমেন্ট মাইগ্রেশন অনুযায়ী রিটার্ন অ্যারে (Definition Setup)

আমাদের কমেন্ট টেবিলের মাইগ্রেশনে যে ফিল্ডগুলো আছে (যেমন: `post_id`, `parent_id`, `author_name`, `author_email`, `user_id`, `content`, `is_approved`, `ip_address`, `user_agent`), সেগুলো সব আমাদের ফ্যাক্টরির রিটার্ন অ্যাসোসিয়েটিভ অ্যারেতে লাগবে। চলুন ধাপে ধাপে লজিকসহ কোডগুলো লিখে ফেলি:

#### ক) `post_id` সেটআপ (Null Coalescing Logic)

এখানে লজিকটা খুব চমৎকার! ডাটাবেজে যদি অলরেডি কোনো পোস্ট থাকে, তবে রেন্ডম অর্ডারে প্রথম পোস্টের আইডিটা নেবে। আর যদি কোনো পোস্ট না থাকে (ডাটাবেজ খালি থাকে), তবে নাল কোয়ালিসিং অপারেটর (`??`) ব্যবহার করে নতুন একটি পোস্ট ফ্যাক্টরি দিয়ে ক্রিয়েট করে তার আইডি নেবে:

```php
'post_id' => Post::inRandomOrder()->first()?->id ?? Post::factory(),

```

#### খ) `parent_id` সেটআপ

প্রথম যখন কোনো একজন ইউজার ফ্রেশ কমেন্ট করে, তখন সেই কমেন্টটার কোনো প্যারেন্ট কমেন্ট থাকে না (সেটি কোনো রিপ্লাই নয়)। এইজন্য প্রাথমিকভাবে আমরা এখানে সরাসরি বলে দিতে পারি `null`:

```php
'parent_id' => null,

```

#### গ) `author_name` এবং `author_email` সেটআপ

এই ফিল্ডগুলোর ডেটা আমরা সরাসরি ফেকার (Faker) থেকে নিয়ে আসবো। নামের জন্য `name()` এবং ইমেইলের জন্য `safeEmail()` মেথড ব্যবহার করবো:

```php
'author_name' => fake()->name(),
'author_email' => fake()->safeEmail(),

```

#### ঘ) `user_id` সেটআপ

পোস্টের মতোই আমরা ইউজার আইডি ডাটাবেজ থেকে রেন্ডম অর্ডারে প্রথম ইউজারকে তুলে নিয়ে আসবো:

```php
'user_id' => User::inRandomOrder()->first()?->id ?? User::factory(),

```

#### ঙ) `content` সেটআপ (Random Element from Array)

কন্টেন্ট জেনারেট করার জন্য আমরা ফেকারের `randomElement()` মেথড ব্যবহার করবো এবং প্যারামিটার হিসেবে উপরে ডিফাইন করা আমাদের কাস্টম `$comments` ভেরিয়েবলটা পাস করে দেবো। এর ফলে আমাদের তৈরি করা বাংলা কমেন্ট অ্যারে থেকে রেন্ডমলি যেকোনো একটা একটা করে কমেন্ট জেনারেট হবে:

```php
'content' => fake()->randomElement($comments),

```

#### চ) `is_approved` সেটআপ (Boolean with Weight)

কমেন্টটি কি এপ্রুভড নাকি পেন্ডিং তা ডিফাইন করার জন্য আমরা ফেকারের `boolean()` মেথড ব্যবহার করবো। আমরা চাই ১০০টি কমেন্টের মধ্যে যেন প্রায় ৮০টি কমেন্ট এপ্রুভড হয় (একটু বেশি রেশিও), তাই আমরা এখানে ওয়েট হিসেবে `80` পাস করবো:

```php
'is_approved' => fake()->boolean(80),

```

#### ছ) `ip_address` সেটআপ

ডামি আইপি এড্রেস জেনারেট করার জন্য আমরা আইপির ভার্সন ৪ ব্যবহার করার চেষ্টা করবো। স্প্যামিং বা ট্র্যাকিং লজিকের জন্য এটি দরকার হয়। ফেকারের `ipv4()` মেথড দিয়ে এটি খুব সহজেই করা যায়:

```php
'ip_address' => fake()->ipv4(),

```

#### জ) `user_agent` সেটআপ

ইউজার এজেন্ট মূলত একজন ইউজারের ফেক ব্রাউজার বা ফেক ডিভাইস বা ওএস (OS) এর ইনফরমেশন জেনারেট করে নিয়ে আসে। এর জন্য আমরা ফেকারের `userAgent()` মেথড ব্যবহার করবো:

```php
'user_agent' => fake()->userAgent(),

```

### ৫. কমেন্ট ফ্যাক্টরির সম্পূর্ণ কোড স্ট্রাকচার

আমাদের কমেন্ট ফ্যাক্টরি ফাইলের সম্পূর্ণ কোড রূপটি নিচে দেওয়া হলো। প্রতিটি অ্যাসোসিয়েটিভ অ্যারের লাইনের শেষে কমা (`,`) ঠিকঠাক দেওয়া হয়েছে কিনা তা চেক করে নেবেন:

```php
<?php

namespace Database\Factories;

use App\Models\Comment;
use App\Models\Post;
use App\Models\User;
use Illuminate\Database\Eloquent\Factories\Factory;

/**
 * @extends \Illuminate\Database\Eloquent\Factories\Factory<\App\Models\Comment>
 */
class CommentFactory extends Factory
{
    /**
     * The name of the factory's corresponding model.
     *
     * @var string
     */
    protected $model = Comment::class;

    /**
     * Define the model's default state.
     *
     * @return array<string, mixed>
     */
    public function definition(): array
    {
        // কাস্টম বাংলা কমেন্ট অ্যারে ডিফাইন
        $comments = [
            'অসাধারণ! এই অসাধারণ আর্টিকেল।',
            'খুবই উপকারী কন্টেন্ট।',
            'ধন্যবাদ, এত সুন্দর করে বুঝিয়েছেন।',
        ];

        return [
            'post_id'      => Post::inRandomOrder()->first()?->id ?? Post::factory(),
            'parent_id'    => null,
            'author_name'  => fake()->name(),
            'author_email' => fake()->safeEmail(),
            'user_id'      => User::inRandomOrder()->first()?->id ?? User::factory(),
            'content'      => fake()->randomElement($comments),
            'is_approved'  => fake()->boolean(80),
            'ip_address'   => fake()->ipv4(),
            'user_agent'   => fake()->userAgent(),
        ];
    }
}

```

### ক্লাস রিভিউ ও আগামী পর্বের পরিকল্পনা:

এই পর্বে আমরা আমাদের `CommentFactory`-র বেসিক ব্লুপ্রিন্ট ও ডেফিনেশন অ্যারে লজিক্যালি এবং অত্যন্ত নিখুঁত উপায়ে সাজিয়ে ফেললাম। বিশেষ করে কীভাবে কাস্টম অ্যারে থেকে রেন্ডম এলিমেন্ট নিতে হয় এবং ডাটাবেজে ডেটা না থাকলে কীভাবে নতুন আইডি জেনারেট করতে হয়, তা আমরা ক্লিয়ারলি জানলাম।

কমেন্ট ফ্যাক্টরির বেসিক ডেফিনেশনের কাজ আমরা এখানেই শেষ করছি। আগামী পর্ব থেকে আমরা আমাদের এই কমেন্ট ফ্যাক্টরির পরবর্তী কাস্টম মেথডগুলো (যেমন: চাইল্ড কমেন্ট বা রিপ্লাই স্টেট) নিয়ে কাজ শুরু করবো।

আজকে এই পর্যন্তই, সবাই প্র্যাকটিস করতে থাকুন। আগামী পর্বে আবার দেখা হচ্ছে!
