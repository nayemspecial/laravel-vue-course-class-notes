# লারাভেল কমেন্ট ফ্যাক্টরি (Comment Factory) সেটআপ ও ডেফিনেশন তৈরি

**প্লাটফর্ম:** প্রযুক্তিপ্লাস একাডেমী (ProjuktiPlus Academy)

**বিষয়:** ৮.৯ কোমেন্ট ফ্যাক্টরির প্রয়োজনীয় কিছু স্টেট নিয়ে আলোচনা, কমেন্ট রিপ্লাই স্টেট, কমেন্ট এপ্রুভড স্টেট, পেন্ডিং স্টেট, রেজিস্টার্ড ইউজার স্টেট এবং ভিজিটর বা গেস্ট স্টেট। 

এই পর্বে আমরা আমাদের কমেন্ট ফ্যাক্টরির স্টেটগুলো বা পরবর্তী যে মেথডগুলো আছে, সেগুলো নিয়ে কাজ শুরু করবো। তার জন্য আমরা আমাদের কোড এডিটরে এসে বেশ কিছু ফাংশন ক্রিয়েট করবো।

লারাভেল ফ্যাক্টরির কাস্টম স্টেট মেথডগুলো আমাদের ডামি ডাটা জেনারেট করার প্রক্রিয়াকে আরও অনেক বেশি ফ্লেক্সিবল এবং লজিক্যাল করতে সাহায্য করে। নিচে প্রতিটি স্টেটের কাজ এবং কোড ইমপ্লিমেন্টেশন বিস্তারিতভাবে দেওয়া হলো:

---

### ১. `reply()` মেথড (কমেন্ট রিপ্লাই স্টেট)

প্রথমে আমরা কমেন্টের রিপ্লাই বা চাইল্ড কমেন্ট নিয়ে কাজ করার জন্য একটি মেথড তৈরি করবো।

* **লজিক:** যদি কোনো কমেন্ট অলরেডি ডাটাবেজে থাকে, তবে তার রেন্ডম আইডিটি নিয়ে এসে `parent_id` হিসেবে সেট করবে।
* **নাল সেফটি (Null Safety):** যদি ডাটাবেজে কোনো কমেন্ট না থাকে, তবে আমাদের অ্যাপ্লিকেশন যেন ক্রাশ না করে, সেজন্য আমরা নাল কোয়ালিসিং অপারেটর (`??`) ব্যবহার করে একটি ফলব্যাক লজিক দেবো। ডাটাবেজে কমেন্ট না থাকলে এটি অটোমেটিক নতুন একটি কমেন্ট ফ্যাক্টরি দিয়ে ক্রিয়েট করে তার আইডি বসিয়ে দেবে।

```php
public function reply(): static
{
    return $this->state(fn (array $attributes) => [
        'parent_id' => Comment::inRandomOrder()->first()?->id ?? Comment::factory(),
    ]);
}

```

*(নোট: মেথডের রিটার্ন টাইপ হিসেবে `static` বলে দেওয়া যেতে পারে। স্টেট মেথডের ভেতরে অ্যাসোসিয়েটিভ অ্যারে রিটার্ন করার জন্য আমরা সবসময় থার্ড ব্র্যাকেট `[]` ব্যবহার করবো এবং স্টেট ব্লকের শেষে সেমিকোলন `;` দিতে হবে।)*

### ২. `approved()` মেথড (এপ্রুভড স্টেট)

কমেন্টটি সরাসরি ডাটাবেজে এপ্রুভড হিসেবে দেখানোর জন্য আমরা এই মেথডটি তৈরি করবো।

* **লজিক:** এটি কল করলে কমেন্ট টেবিলের `is_approved` ফিল্ডের ভ্যালু সরাসরি `true` হয়ে যাবে।

```php
public function approved(): static
{
    return $this->state(fn (array $attributes) => [
        'is_approved' => true,
    ]);
}

```

---

### ৩. `pending()` মেথড (পেন্ডিং স্টেট)

যে কমেন্টগুলো অ্যাডমিন প্যানেল থেকে এখনো এপ্রুভ করা হয়নি বা পেন্ডিং অবস্থায় আছে, সেগুলোর জন্য এই মেথডটি কাজ করবে।

* **লজিক:** এখানে এসে আমরা জাস্ট ফিল্ডের ভ্যালু চেঞ্জ করে দেবো। অর্থাৎ, `is_approved` ফিল্ডের ভ্যালু সরাসরি `false` করে দেবো।

```php
public function pending(): static
{
    return $this->state(fn (array $attributes) => [
        'is_approved' => false,
    ]);
}

```

---

### ৪. `fromUser()` মেথড (রেজিস্টার্ড ইউজার স্টেট)

যখন আমাদের সিস্টেমে লগইন থাকা কোনো রেজিস্টার্ড ইউজার কমেন্ট করবে, তখন এই স্টেটটি ব্যবহার করা হবে।

* **লজিক:** `user_id` ফিল্ডে ডাটাবেজ থেকে রেন্ডম অর্ডারে প্রথম ইউজারের আইডি এনে বসাবে। যদি কোনো ইউজার ডাটাবেজে না থাকে, তবে ফলব্যাক হিসেবে নতুন ইউজার ক্রিয়েট করে নেবে (`User::factory()`)।
* **অতিরিক্ত কারেকশন:** যদি কোনো রেজিস্টার্ড ইউজার কমেন্ট করে, তবে সেখানে আলাদা করে গেস্ট বা ভিজিটরের নাম (`author_name`) এবং ইমেইল (`author_email`) রাখার কোনো লজিক্যাল প্রয়োজনীয়তা নেই। তাই এই দুটা ফিল্ডকে আমরা এখানে সুন্দর করে `null` সেট করে দেবো।

```php
public function fromUser(): static
{
    return $this->state(fn (array $attributes) => [
        'user_id' => User::inRandomOrder()->first()?->id ?? User::factory(),
        'author_name' => null,
        'author_email' => null,
    ]);
}

```

### ৫. `guest()` মেথড (ভিজিটর বা গেস্ট স্টেট)

রেজিস্টার্ড ইউজারের ঠিক বিপরীত লজিক কাজ করবে এই গেস্ট বা আনলগইনড ভিজিটর কমেন্টের ক্ষেত্রে।

* **লজিক:** যেহেতু এটি গেস্ট কমেন্ট, তাই লজিক্যালি এখানে কোনো রেজিস্টার্ড ইউজারের আইডি থাকবে না। এই কারণে আমরা `user_id` ফিল্ডের ভ্যালু সরাসরি করে দেবো `null`।
* **গেস্ট ইনফো:** ইউজার আইডি নাল হলেও গেস্টের নাম এবং ইমেইল আমাদের লাগবে। তাই আমরা ফেকার (Faker) ব্যবহার করে ডামি নাম এবং সেফ ইমেইল জেনারেট করে দেবো।

```php
public function guest(): static
{
    return $this->state(fn (array $attributes) => [
        'user_id' => null,
        'author_name' => fake()->name(),
        'author_email' => fake()->safeEmail(),
    ]);
}

```

### কমেন্ট ফ্যাক্টরি ক্লাসের সম্পূর্ণ কোড রূপরেখা

সবগুলো মেথড একসাথে সাজানোর পর আমাদের কমেন্ট ফ্যাক্টরি ফাইলটির ভেতরের স্টেট স্ট্রাকচারটি দেখতে এইরকম হবে:

```php
<?php

namespace Database\Factories;

use App\Models\Comment;
use App\Models\Post;
use App\Models\User;
use Illuminate\Database\Eloquent\Factories\Factory;

/**
 * @extends \Illuminate\Database\Eloquent\Factories\Factory<\App\Models\Comment>
 */
class CommentFactory extends Factory
{
    /**
     * Define the model's default state.
     *
     * @return array<string, mixed>
     */
    public function definition(): array
    {
        $comments = [
            'অসাধারণ! এই অসাধারণ আর্টিকেল।',
            'খুবই উপকারী কন্টেন্ট।',
            'ধন্যবাদ, এত সুন্দর করে বুঝিয়েছেন।',
        ];

        return [
            'post_id' => Post::inRandomOrder()->first()?->id ?? Post::factory(),
            'parent_id' => null,
            'author_name' => fake()->name(),
            'author_email' => fake()->safeEmail(),
            'user_id' => User::inRandomOrder()->first()?->id ?? User::factory(),
            'content' => fake()->randomElement($comments),
            'is_approved' => fake()->boolean(80),
            'ip_address' => fake()->ipv4(),
            'user_agent' => fake()->userAgent(),
        ];
    }

    /**
     * Indicate that the comment is a reply.
     */
    public function reply(): static
    {
        return $this->state(fn (array $attributes) => [
            'parent_id' => Comment::inRandomOrder()->first()?->id ?? Comment::factory(),
        ]);
    }

    /**
     * Indicate that the comment is approved.
     */
    public function approved(): static
    {
        return $this->state(fn (array $attributes) => [
            'is_approved' => true,
        ]);
    }

    /**
     * Indicate that the comment is pending.
     */
    public function pending(): static
    {
        return $this->state(fn (array $attributes) => [
            'is_approved' => false,
        ]);
    }

    /**
     * Indicate that the comment is from a registered user.
     */
    public function fromUser(): static
    {
        return $this->state(fn (array $attributes) => [
            'user_id' => User::inRandomOrder()->first()?->id ?? User::factory(),
            'author_name' => null,
            'author_email' => null,
        ]);
    }

    /**
     * Indicate that the comment is from a guest.
     */
    public function guest(): static
    {
        return $this->state(fn (array $attributes) => [
            'user_id' => null,
            'author_name' => fake()->name(),
            'author_email' => fake()->safeEmail(),
        ]);
    }
}

```

### সারাংশ ও পরবর্তী পরিকল্পনা

এই পর্যন্তই হচ্ছে আমাদের কমেন্ট ফ্যাক্টরি নিয়ে মূলত কাজ। আমাদের প্রজেক্টের জন্য যে কয়টি গুরুত্বপূর্ণ টেবিল ছিল, সবগুলোর ডাটাই এখন আমরা ফ্যাক্টরি দিয়ে সাকসেসফুলি জেনারেট করতে পারবো। আমরা ইতিমধ্যে বেশ কয়েকটি গুরুত্বপূর্ণ ফ্যাক্টরি ফাইল ক্রিয়েট করে ফেলেছি:

1. **UserFactory**
2. **TagFactory**
3. **PostFactory**
4. **CommentFactory**
5. **CategoryFactory**

যেহেতু আমাদের খুবই গুরুত্বপূর্ণ ফ্যাক্টরি ফাইলগুলো ক্রিয়েট করার কাজ অলরেডি কমপ্লিট, তাই এর পর আমরা যেটি করতে পারি তা হলো—আমরা **সিডার (Seeder)** নিয়ে কাজ শুরু করতে পারি।

আর সিডার নিয়ে কাজ শুরু করতে হলে প্রথমেই তো আমাদের যেতে হয় হচ্ছে `UserSeeder`-এ। আমরা এই পর্বটা এখানেই শেষ করছি। পরবর্তী পর্বে আমরা সিডার ফাইল নিয়ে কাজ শুরু করবো।
