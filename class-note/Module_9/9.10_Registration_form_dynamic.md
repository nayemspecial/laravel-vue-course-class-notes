**প্রযুক্তি প্লাস একাডেমির** পক্ষ থেকে আপনাদের সবাইকে স্বাগতম জানাচ্ছি। গত পর্বে আমরা আমাদের রেজিস্ট্রেশন ফর্মের জন্য ব্লেড ফাইল এবং বেসিক স্ট্রাকচার তৈরি করেছিলাম। এই পর্বে আমরা আমাদের রেজিস্ট্রেশন ফর্মের কোডগুলোকে পুরোপুরি **ডায়নামিক করব**।

আপনাদের বোঝার সুবিধার্থে আজকের ক্লাসের সম্পূর্ণ আলোচনা, কোড স্ট্রাকচার ও লজিক্যাল ফ্লো নিচে সুন্দরভাবে সাজিয়ে দেওয়া হলো:

## ১. এইচটিএমএল ফর্ম ট্যাগে অ্যাট্রিবিউট যুক্ত করা (Form Attributes)

প্রথমেই আমরা চলে যাব আমাদের রেজিস্ট্রেশন ব্লেড ফাইলে (`auth/register.blade.php`)। সেখানে থাকা `<form>` ট্যাগটিতে দুটি গুরুত্বপূর্ণ অ্যাট্রিবিউট এবং সিকিউরিটি টোকেন বসিয়ে দেব:

* **Action:** ডাটা সাবমিট করার রাউট বলে দেব—`action="{{ route('register') }}"`।


* **Method:** যেহেতু ডাটা সাবমিট হবে, তাই মেথড হবে—`method="POST"`।


* **CSRF Token:** লারাভেলের সিকিউরিটির জন্য ফর্ম ট্যাগের ঠিক নিচেই `@csrf` লিখে দেব।

## ২. ইনপুট ফিল্ডের নেম (Name) ভেরিফিকেশন

ফর্মে থাকা ইনপুট ফিল্ডগুলোর `name` অ্যাট্রিবিউটগুলো ঠিক আছে কিনা তা চেক করে নিতে হবে:

* নাম ফিল্ডের জন্য: `name="name"`

* ইমেইল ফিল্ডের জন্য: `name="email"`

* পাসওয়ার্ড ফিল্ডের জন্য: `name="password"`

* কনফার্ম পাসওয়ার্ড ফিল্ডের জন্য: `name="password_confirmation"`


> **গুরুত্বপূর্ণ নোট:** কনফার্ম পাসওয়ার্ডের ইনপুট ফিল্ডে হুবহু **`password_confirmation`** কিওয়ার্ডটাই নাম (name) হিসেবে দিতে হবে। এখানে শুধু `password` বা অন্য কিছু দিলে লারাভেলের কনফার্মেশন ভ্যালিডেশন কাজ করবে না।

## ৩. পোস্ট রাউট তৈরি (Post Route Setup)

ডাটা সাবমিট করার জন্য আমাদের `routes/web.php` ফাইলে একটি পোস্ট (`POST`) রাউট তৈরি করতে হবে। আমরা গেট রাউটের নিচেই এটি ডিফাইন করছি:

```php
Route::post('/register', [RegisterController::class, 'register']);

```

## ৪. কন্ট্রোলারে রেজিস্ট্রেশন ও ভ্যালিডেশন লজিক (Register Logic)

এবার আমরা চলে যাব আমাদের `RegisterController.php` ফাইলে। সেখানে আমরা `register` নামের মেথডটি তৈরি করব এবং ইউজারের পাঠানো ডাটা ভ্যালিডেশন করে ডাটাবেজে সেভ করার লজিক লিখব:

```php
<?php

namespace App\Http\Controllers\Auth;

use App\Http\Controllers\Controller;
use App\Models\User;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Auth;
use Illuminate\Support\Facades\Hash;

class RegisterController extends Controller
{
    // রেজিস্ট্রেশন ফর্ম দেখানোর মেথড
    public function showRegisterForm()
    {
        return view('auth.register');
    }

    // রেজিস্ট্রেশন ডাটা প্রসেস করার মেথড
    public function register(Request $request)
    {
        // ১. ইনপুট ডাটা ভ্যালিডেশন করা
        $validatedData = $request->validate([
            'name'     => 'required|string|max:255',
            'email'    => 'required|string|email|max:255|unique:users',
            'password' => 'required|string|min:6|confirmed',
        ]);

        // ২. ডাটাবেজে নতুন ইউজার তৈরি করা
        $user = User::create([
            'name'     => $validatedData['name'],
            'email'    => $validatedData['email'],
            'password' => Hash::make($validatedData['password']), // পাসওয়ার্ড হ্যাশ করা
            'role'     => 'reader', // নতুন রেজিস্ট্রেশন করা সবাই ডিফল্টভাবে রিডার হবে
        ]);

        // ৩. ইউজারকে অটোমেটিক লগইন করানো (সেশন শুরু করা)
        Auth::login($user);

        // ৪. সফলভাবে রেজিস্ট্রেশন শেষে রিডার বা নরমাল ড্যাশবোর্ডে রিডাইরেক্ট করা
        return redirect()->route('dashboard');
    }
}

```

### লজিকগুলোর গুরুত্বপূর্ণ ব্যাখ্যা:

* **`confirmed` ভ্যালিডেশন:** লারাভেলের এই রুলটি অটোমেটিক `password` ফিল্ডের সাথে `password_confirmation` ফিল্ডের মান মিলিয়ে দেখে।


* **`Hash::make()`:** ব্যবহারকারীর পাসওয়ার্ডটি সরাসরি ডাটাবেজে সেভ না করে সিকিউরিটির জন্য হ্যাশ (এনক্রিপ্ট) করে সেভ করে।


* **`role => 'reader'`:** সিকিউরিটির স্বার্থে নতুন অ্যাকাউন্ট খোলা যে কেউ যেন এডমিন হতে না পারে, তাই ডিফল্ট রোল 'reader' সেট করা হয়েছে।


* **`Auth::login($user)`:** এখানে `Auth::attempt()` ব্যবহার না করে সরাসরি `Auth::login()` ব্যবহার করা হয়েছে। এটি ইমেইল-পাসওয়ার্ড পুনরায় যাচাই না করেই নতুন তৈরি হওয়া বৈধ ইউজারটির জন্য সরাসরি একটি সেশন শুরু করে দেয়।

## ৫. প্র্যাকটিক্যাল টেস্ট ও যাচাইকরণ

কোড লেখার পর আমরা ব্রাউজারে আমাদের ফর্মটি পূরণ করব (নাম, ইমেইল এবং পাসওয়ার্ড ও কনফার্ম পাসওয়ার্ড হিসেবে 'password' লিখে) এবং **Register** বাটনে ক্লিক করব। যদি সবকিছু ঠিক থাকে, তবে দেখতে পাবেন ডাটাবেজে সফলভাবে নতুন ইউজার তৈরি হয়ে গেছে এবং ইউজার অটো-লগইন হয়ে সরাসরি ড্যাশবোর্ডে রিডাইরেক্ট হয়ে চলে গেছে!

আমাদের রেজিস্ট্রেশনের ব্যাকএন্ড লজিক নিখুঁতভাবে কাজ করছে। তবে কোনো ইউজার যদি ফর্মে ভুল তথ্য ইনপুট দেয় (যেমন: ইমেইল ফরম্যাট ভুল করা বা ৬ অক্ষরের কম পাসওয়ার্ড দেওয়া), তবে তাকে স্ক্রিনে সুন্দরভাবে সতর্কবার্তা দেখানোর জন্য আমাদের ফর্ম ভ্যালিডেশন মেসেজ নিয়ে কাজ করতে হবে। সেই কাজটি আমরা আগামী পর্বে সম্পন্ন করব।

আজকের ক্লাসটি এখানেই শেষ করছি। সবাই ভালোমতো প্র্যাকটিস করুন। ধন্যবাদ!
