**হ্যালো এভরিওয়ান!** প্রযুক্তি প্লাস একাডেমির পক্ষ থেকে আপনাদের সবাইকে স্বাগতম জানাচ্ছি।

আজকের ক্লাসে আমরা দেখবো কীভাবে আমাদের প্রোজেক্টে **লগইন ফিচার (Login Feature)** বা লগইন ফর্ম যুক্ত করতে হয় এবং এর জন্য প্রয়োজনীয় ব্যাক-এন্ড লজিক সেটআপ করতে হয়। আজকের ক্লাসের মূল আলোচনা ও প্র্যাকটিক্যাল নোট নিচে দেওয়া হলো:

## ১. ভিউ ফোল্ডার এবং ফাইল তৈরি

প্রথমেই আমরা আমাদের লগইন ফর্মটি দেখানোর জন্য ভিউ ফাইল তৈরি করবো।

* **ফোল্ডার তৈরি:** `resources/views` ফোল্ডারের ভেতরে অথেন্টিকেশন রিলেটেড কাজের জন্য `auth` নামে একটি নতুন ফোল্ডার তৈরি করি.


* **ফাইল তৈরি:** এই `auth` ফোল্ডারের ভেতরে `login.blade.php` নামে একটি ব্লেড ফাইল তৈরি করি.



---

## ২. রাউট (Route) সেটআপ এবং নেম রাউটের গুরুত্ব

লগইন পেজটি ব্রাউজারে শো করানোর জন্য আমরা `routes/web.php` ফাইলে একটি গেট রাউট (Get Route) তৈরি করবো:

```php
Route::get('/login', [LoginController::class, 'showLoginForm'])->name('login');

```

### নেম রাউট (`name('login')`) কেন ব্যবহার করবো?

ভবিষ্যতে যদি ক্লায়েন্ট বা অ্যাডমিন বলে যে সে ইউআরএলটি পরিবর্তন করে `/admin-login` বা অন্য কিছু রাখতে চায়, তখন আমরা সহজেই তা করতে পারবো। প্রোজেক্টের বাকি যেসব জায়গায় এই লিঙ্ক ব্যবহার করা হয়েছে, সেখানে কোনো কোড চেঞ্জ করা লাগবে না; নেম রাউট নিজেই সবকিছু হ্যান্ডেল করবে।

## ৩. কন্ট্রোলার ও মেথড তৈরি

রাউটের কাজ শেষ হলে আমাদের `LoginController` তৈরি করতে হবে।

### কন্ট্রোলার তৈরির কমান্ড:

টার্মিনালে গিয়ে আমরা পিএইচপি আর্টিজান কমান্ড ব্যবহার করে অথেন্টিকেশন ফোল্ডারের ভেতরে কন্ট্রোলারটি তৈরি করবো:

```bash
php artisan make:controller Auth/LoginController

```

কমান্ডটি রান করলে `app/Http/Controllers/Auth` ফোল্ডারের ভেতরে `LoginController.php` ফাইলটি তৈরি হবে।

### ওয়েব ডট পিএইচপিতে কন্ট্রোলার ইমপোর্ট করা:

কন্ট্রোলারটি তৈরি করার পর `web.php` ফাইলের ওপরে এটি অবশ্যই অ্যাড বা ইমপোর্ট করে নিতে হবে, না হলে এরর আসবে:

```php
use App\Http\Controllers\Auth\LoginController;

```

### কন্ট্রোলারে মেথড তৈরি:

এবার `LoginController` ক্লাসের ভেতরে `showLoginForm` মেথডটি তৈরি করে আমাদের তৈরি করা ব্লেড ভিউ ফাইলটি রিটার্ন করে দেবো:

```php
public function showLoginForm()
{
    return view('auth.login');
}

```

## ৪. লগইন ফর্ম ডাইনামিক করা

লগইন পেজের স্ট্যাটিক এইচটিএমএল কোডগুলো `login.blade.php` ফাইলে নেওয়ার পর লারাভেলের ব্যাক-এন্ডের সাথে যুক্ত করার জন্য ফর্মে কিছু গুরুত্বপূর্ণ পরিবর্তন করতে হবে:

### ক. ফর্ম ট্যাগ (Form Tag) আপডেট:

ফর্ম ট্যাগের ভেতরে `action` এবং `method` অ্যাট্রিবিউট দুটি অবশ্যই যুক্ত করতে হবে:

```html
<form action="{{ route('login') }}" method="POST">

```

* **অ্যাকশন (Action):** সাবমিট করার পর ডেটা কোথায় যাবে তা নেম রাউটের মাধ্যমে বলে দেওয়া হলো।


* **মেথড (Method):** ডেটা সার্ভারে পাঠানোর জন্য মেথডটি `POST` করে দেওয়া হলো।



### খ. সিএসআরএফ টোকেন (CSRF Token):

লারাভেলে ফর্ম সাবমিশনের ক্ষেত্রে সিকিউরিটির জন্য **CSRF Token** অত্যন্ত গুরুত্বপূর্ণ। ফর্ম ট্যাগের ঠিক নিচের লাইনেই `@csrf` ডিরেক্টিভটি যুক্ত করতে হবে:

```html
@csrf

```

> **নোট:** এটি না দিলে লারাভেলে `419` এরর আসবে, যা একটি যথেষ্ট ঝামেলাযুক্ত এরর।
> 
> 

### গ. ইনপুট ফিল্ডের নাম (Input Name Attribute):

লারাভেলের ব্যাক-এন্ডে ডেটা রিসিভ করার জন্য প্রতিটি ইনপুট ফিল্ডে অবশ্যই `name` অ্যাট্রিবিউট থাকতে হবে:

* **ইমেইল ফিল্ডের জন্য:** `name="email"`

* **পাসওয়ার্ড ফিল্ডের জন্য:** `name="password"`
