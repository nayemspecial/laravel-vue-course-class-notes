## ১. পোস্ট রাউট (Post Route) সেটআপ

গত পর্বে আমরা লগইন ফর্ম দেখানোর জন্য গেট রাউট তৈরি করেছিলাম। এবার ফর্ম সাবমিট করার জন্য `routes/web.php` ফাইলে গেট রাউটটিকে ডুপ্লিকেট করে মেথডটি `POST` করে দেবো:

```php
Route::post('/login', [LoginController::class, 'login']);

```

* **নোট:** এখানে পোস্ট রাউটের জন্য আলাদা করে আর `name()` রাউট ব্যবহার করার প্রয়োজন নেই।



---

## ২. কন্ট্রোলারে লগইন মেথড ও রিকোয়েস্ট বাইন্ডিং

এবার `LoginController`-এ গিয়ে ডেটা প্রসেস করার জন্য `login` নামে একটি নতুন মেথড তৈরি করতে হবে। সাবমিট করা ডেটা ধরার জন্য মেথডের প্যারামিটারে লারাভেলের `Request` ক্লাসটি মেথড বাইন্ডিং করে নেবো:

```php
public function login(Request $request)
{
    // এখানে আমাদের যাবতীয় লগইন লজিক থাকবে
}

```

> **টিপস:** ফর্ম থেকে ইমেইল, পাসওয়ার্ড এবং সিএসআরএফ টোকেন ঠিকমতো আসছে কিনা তা প্রাথমিকভাবে চেক করার জন্য আমরা মেথডের ভেতরে `dd($request->all());` (Die and Dump) ব্যবহার করে পরীক্ষা করতে পারি।
> 
>

## ৩. ডেটা ভ্যালিডেশন (Validation)

সার্ভারে ডেটা প্রসেস করার আগে অবশ্যই তা ভ্যালিডেশন করে নিতে হবে। এর জন্য আমরা একটি অ্যাসোসিয়েটিভ অ্যারো (Associative Array) ব্যবহার করে নিয়ম ঠিক করে দেবো:

```php
$credentials = $request->validate([
    'email' => ['required', 'email'],
    'password' => ['required'],
]);

```

এখানে আমরা ইমেইল এবং পাসওয়ার্ড উভয় ফিল্ডকেই `required` (বাধ্যতামূলক) করলাম এবং ইমেইলের ফরম্যাট ঠিক আছে কিনা তা যাচাইয়ের নিয়ম বলে দিলাম。

---

## ৪. অথেন্টিকেশন ও সেশন জেনারেট করা

ভ্যালিডেশন পাস করার পর লারাভেলের বিল্ট-ইন `Auth` ফ্যাসাডের `attempt` মেথড দিয়ে ইউজারকে লগইন করানোর চেষ্টা করা হবে। কাজ শুরু করার আগে কন্ট্রোলার ফাইলের একদম ওপরে `Auth` ফ্যাসাডটি ইমপোর্ট বা সেটআপ করে নিতে হবে:

```php
use Illuminate\Support\Facades\Auth;

```

লগইন প্রসেসটি আমরা একটি `if` কন্ডিশনের ভেতরে রাখবো। ক্রিডেনশিয়াল মিলে গেলে সিকিউরিটির জন্য একটি নতুন সেশন রিজেনারেট (Session Regenerate) করা হবে:

```php
if (Auth::attempt($credentials)) {
    $request->session()->regenerate();
    
    // রোল চেকিং এর কাজ এখানে হবে
}

```

## ৫. রোল ভিত্তিক ড্যাশবোর্ডে রিডাইরেক্ট (Role-based Redirection)

লগইন করা ইউজারদের মধ্যে বিভিন্ন রোল থাকতে পারে (যেমন: অ্যাডমিন, সুপার অ্যাডমিন কিংবা সাধারণ ছাত্র)। তাই সেশন জেনারেট হওয়ার পর আমরা ইউজারের রোল চেক করবো:

```php
$user = Auth::user();

if ($user->role === 'admin' || $user->role === 'super-admin') {
    return redirect()->intended(route('admin.dashboard'));
}

// অন্যথায় সাধারণ ইউজার/ছাত্রদের ড্যাশবোর্ডে যাবে
return redirect()->intended(route('dashboard'));

```

* **অর অপারেটর (`||`):** এর মাধ্যমে চেক করা হচ্ছে যে ইউজারের রোল যদি `admin` অথবা `super-admin` হয়, তবে তাকে সরাসরি অ্যাডমিন ড্যাশবোর্ডে রিডাইরেক্ট করা হবে।


* **`intended()` মেথড:** ইউজার লগইন করার আগে যে পেজে যাওয়ার চেষ্টা করছিলেন, লগইন সফল হওয়ার পর তাকে সরাসরি সেই পেজেই নিয়ে যাওয়ার জন্য এটি ব্যবহৃত হয়।



---

**আগামী পর্বের পড়া:** আমাদের ব্যাক-এন্ডের লগইন ও রোল চেকিং লজিক সফলভাবে সম্পন্ন হয়েছে। ইউজার লগইন হওয়ার পর যে ড্যাশবোর্ডে পৌঁছাবে, সেখানকার ডিজাইন এবং পরবর্তী ইন্টারফেস কেমন হবে, তা আমরা আগামী পর্বে বিস্তারিত শিখবো। সবাই এই কোডটুকু নিজেদের প্রজেক্টে লিখে রেডি রাখবে।
