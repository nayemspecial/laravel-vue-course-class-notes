**বিষয়:** ৯.৭ মিডিলওয়্যার কি, কাস্টম মিডিলওয়্যার তৈরি, মিডিলওয়্যারে রোল চেকিং লজিক, মিডিলওয়্যার রেজিস্টার করা, রাউটে মিডিলওয়্যার অ্যাপ্লাই করা, প্র্যাকটিক্যাল টেস্ট ও যাচাইকরণ।

**প্রযুক্তি প্লাস একাডেমির** পক্ষ থেকে আপনাদের সবাইকে স্বাগতম জানাচ্ছি। আজকে আমরা যেটি করব সেটি হচ্ছে, আমরা আমাদের ওয়েবসাইটে একটা ফিল্টারিং করব। যাতে করে একজন রিডার যেন এডমিন প্যানেলে না যেতে পারে। তার মানে, রোল-ভিত্তিক যে ভিজিটিং সিস্টেম বা অ্যাক্সেস কন্ট্রোল, সেটি আমরা ক্রিয়েট করব।

আজকের ক্লাসের সম্পূর্ণ আলোচনা, কোড স্ট্রাকচার ও লজিক্যাল ফ্লো নিচে সুন্দরভাবে সাজিয়ে দেওয়া হলো:

## ১. সমস্যার বাস্তব চিত্র ও সমাধান ধারণা

প্রথমে আমরা সমস্যাটি প্র্যাক্টিক্যালি দেখে নিই। আমরা যদি একজন সাধারণ রিডারের অ্যাকাউন্ট থেকে লগইন করি (লগইন পাসওয়ার্ড দিয়ে) এবং লগইন করার পর ব্রাউজারের ইউআরএল-এ সরাসরি `/admin/dashboard` লিখে এন্টার প্রেস করি, তাহলে দেখতে পাব যে একজন রিডারও এডমিন ড্যাশবোর্ডে চলে যাচ্ছে। এটি তো সিকিউরিটির দিক থেকে একদমই ঠিক না, তাই না?

লারাভেলে এই ধরনের সিকিউরিটি ফিল্টারিং বা অ্যাক্সেস কন্ট্রোলের কাজটি অত্যন্ত চমৎকারভাবে সমাধান করে **মিডিলওয়্যার (Middleware)**। আমরা এখন একটি কাস্টম মিডিলওয়্যার তৈরি করে এই সমস্যার সমাধান করব।

---

## ২. কাস্টম মিডিলওয়্যার তৈরি (Creating Middleware)

মিডিলওয়্যার তৈরি করার জন্য আমরা কোড এডিটর ওপেন করে একটি নতুন টার্মিনাল নেব এবং কারিগরী কমান্ডটি রান করব:

```bash
php artisan make:middleware AdminMiddleware

```

এই কমান্ডটি রান করার সাথে সাথে আমাদের প্রোজেক্টের `app/Http/Middleware` ডিরেক্টরিতে `AdminMiddleware.php` নামে একটি নতুন ক্লাস ফাইল তৈরি হয়ে যাবে।

---

## ৩. মিডিলওয়্যারে রোল চেকিং লজিক (Authorization Logic)

এবার আমরা আমাদের তৈরি করা `AdminMiddleware.php` ফাইলটি ওপেন করব এবং এর `handle` মেথডের ভেতরে আমাদের ফিল্টারিং লজিকটি লিখব।

এখানে আমাদের কন্ডিশন হবে: **যদি ভিজিটর লগইন করা থাকে এবং তার রোল যদি `admin` না হয় এবং একই সাথে তার রোল যদি `super-admin`-ও না হয়, তবে তাকে রিডাইরেক্ট করে সাধারণ ইউজার ড্যাশবোর্ডে পাঠিয়ে দেওয়া হবে।**

```php
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Auth;
use Symfony\Component\HttpFoundation\Response;

class AdminMiddleware
{
    /**
     * Handle an incoming request.
     */
    public function handle(Request $request, Closure $next): Response
    {
        // ইউজার লগইন অবস্থায় আছে কিনা এবং সে এডমিন বা সুপার-এডমিন কোনোটাই নয় কিনা তা চেক করা
        if (Auth::check() && Auth::user()->role !== 'admin' && Auth::user()->role !== 'super-admin') {
            
            // কন্ডিশন সত্য হলে তাকে রিডার বা নরমাল ড্যাশবোর্ডে রিডাইরেক্ট করা হবে
            return redirect()->route('dashboard');
        }

        // কন্ডিশন মিথ্যা হলে পরবর্তী রিকোয়েস্টের অনুমতি দেওয়া হবে
        return $next($request);
    }
}

```

> **লজিক বিশ্লেষণ:** যখন কোনো এডমিন বা সুপার-এডমিন লগইন করবেন, তখন `if` কন্ডিশনটি মিথ্যা হয়ে যাবে এবং নিচের `return $next($request)` লাইনটি এক্সিকিউট হবে। এর অর্থ হচ্ছে—তাকে আর আটকানো হবে না, সে এডমিন ড্যাশবোর্ডে প্রবেশ করতে পারবে।
> 
>

## ৪. লারাভেল এ মিডিলওয়্যার রেজিস্টার করা (Middleware Registration)

লারাভেল ১১ (Laravel 11) ভার্সন থেকে মিডিলওয়্যার রেজিস্টার করার নিয়মে কিছুটা পরিবর্তন এসেছে। এখন যেকোনো কাস্টম মিডিলওয়্যার রেজিস্টার করতে হয় বুটস্ট্রাপ ফোল্ডারের `app.php` ফাইলে।

এর জন্য আমরা `bootstrap/app.php` ফাইলটি ওপেন করব এবং `withMiddleware` অংশের ভেতরে আমাদের তৈরি করা মিডিলওয়্যারটির একটি অ্যালিয়াস (Alias) বা ডাকনাম ডিফাইন করে দেব:

```php
use App\Http\Middleware\AdminMiddleware;

->withMiddleware(function (Middleware $middleware) {
    $middleware->alias([
        'admin' => AdminMiddleware::class,
    ]);
})

```

(মনে রাখবেন, অ্যাসোসিয়েটিভ অ্যারে শেষ করার সময় কমা `,` দিতে ভুলবেন না, অন্যথায় সিনট্যাক্স এরর দেখাতে পারে)

## ৫. রাউটে মিডিলওয়্যার অ্যাপ্লাই করা (Applying Middleware to Routes)

মিডিলওয়্যার রেজিস্টার করার পর আমাদের মেইন কাজ হচ্ছে এটিকে রাউটে অ্যাপ্লাই বা সেটআপ করা। আমরা আমাদের `routes/web.php` ফাইলটিতে যাব এবং লগইন ও রোল ভেরিফিকেশনের ওপর ভিত্তি করে রাউটগুলোকে গ্রুপ করে নেব:

```php
use App\Http\Controllers\Auth\LoginController;

// ১. শুধুমাত্র লগইন করা (Authenticated) ইউজারদের জন্য গ্রুপ রাউট
Route::middleware(['auth'])->group(function () {
    
    // নরমাল ড্যাশবোর্ড রাউট
    Route::get('/dashboard', function () {
        return view('dashboard');
    })->name('dashboard');
    
});

// ২. লগইন করা এবং একই সাথে 'Admin' রোল বিশিষ্ট ইউজারদের জন্য গ্রুপ রাউট
Route::middleware(['auth', 'admin'])->group(function () {
    
    // এডমিন ড্যাশবোর্ড রাউট
    Route::get('/admin/dashboard', function () {
        return view('admin.dashboard');
    })->name('admin.dashboard');
    
});

```

## ৬. প্র্যাকটিক্যাল টেস্ট ও যাচাইকরণ

এবার আমরা ব্রাউজারে গিয়ে পেজটি রিফ্রেশ করব। এখন যদি আমরা একজন সাধারণ রিডার বা ইউজার হিসেবে লগইন থাকা অবস্থায় জোরপূর্বক ইউআরএল-এ `/admin/dashboard` লিখে প্রবেশ করতে চাই, তবে দেখতে পাব যে মিডিলওয়্যার আমাদের আটকে দিচ্ছে এবং রিডাইরেক্ট করে পুনরায় আমাদের সাধারণ ইউজার ড্যাশবোর্ডেই ফিরিয়ে নিয়ে আসছে!

আমাদের রোল-ভিত্তিক সিকিউরিটি ফিল্টারিং সিস্টেমটি সফলভাবে কাজ করছে। পরবর্তী পর্বে আমরা ইউজার এক্সপেরিয়েন্স আরও সুন্দর করার জন্য ফর্মে কীভাবে ডায়নামিক এরর মেসেজ বা অ্যালার্ট শো করাতে হয় তা নিয়ে বিস্তারিত আলোচনা করব।

আজকের পর্বটি এখানেই শেষ করছি, সবাই ভালো থাকবেন। ধন্যবাদ!
